问当地小程序需求分析说明（二期）

【Slogan】当地想问啥，就来问当地

【版本说明】二期版本在一期MVP基础上进行功能优化和体验提升，增加举报、评价系统优化、消息管理等功能

一、一期功能回顾

一期MVP版本已完成核心功能：
• 微信授权登录 + 新用户100积分
• 发布求助（地点、类型、描述、有效期、悬赏积分）
• 任务匹配 + 微信服务通知
• 帮助者接单 + 任务列表
• 聊天功能（文字、图片）
• 确认完成 + 积分解冻/到账
• 星级评价（1-5星）
• 任务有效期管理（超时自动取消）
• 积分明细查看
• 个人信息完善
• 签到功能（连续签到递增）
• 邀请好友（双方各得50积分）
• 实时消息推送（watch+轮询双方案）
• 退出登录功能
• 帮助者资料完善（意愿/地点/类型设置）
• 任务大厅个性化推荐（根据常去地点和类型）
• 聊天图片自适应显示
• 评价系统基础（评分计算）

二、二期新增/优化功能

2.1 任务聊天超时自动完成

需求描述：
• 任务聊天超过12小时时，系统提示用户聊天时间较长
• 双方都停止发消息6小时后，给求助者发送系统消息通知，提示求助者是否确认任务完成
• 双方都停止发消息12小时后，任务自动完成（状态改为"已完成"）
• 若是系统自动进行任务完成时，系统自动给帮助者5星评价
• 任务完成时，双方都会收到系统消息通知

2.3 我的求助列表左滑删除

需求描述：
• 在"我的求助"列表页，每个任务卡片可以左滑
• 左滑后显示"删除"按钮
• 点击删除需要二次确认
• 删除为"假删除"，仅求助者自己看不见
• 帮助者在"已接任务"列表中仍能看到该任务

2.4 消息中心消息置顶和删除

需求描述：
• 在消息中心页面，每条消息可以左滑
• 左滑后显示"删除"和"置顶"两个按钮
• 删除有二次确认提示："删除后将无法找回"
• 置顶是将消息固定在列表顶部
• 可以置顶多条消息，置顶的消息按置顶时间排序

2.5 聊天页面举报功能

需求描述：
• 在聊天页面，求助者可以举报帮助者
• 举报流程：
  1. 点击举报按钮
  2. 选择举报类型
  3. 填写举报事由（必填）
  4. 提交截图（可选，最多3张）
  5. 提示："客服会查看当前聊天记录，是否同意？"同意才能提交
  6. 点击提交时，提示："将自动结束任务，由客服判定积分归属"
  7. 点击确定后，跳转到评价页面
• 举报类型（可选项）：
  - 虚假信息（帮助者提供的信息与实际不符）
  - 态度恶劣（使用侮辱性语言或威胁）
  - 骚扰行为（频繁发送无关信息）
  - 违规交易（要求私下转账或交易）
  - 其他（补充说明）
• 举报后任务状态变为"已举报"

2.6 接单限制规则

需求描述：
• 接单限制规则（在用户协议中明确）：
  - 评分 ≥ 4星：无限制
  - 3星 ≤ 评分 < 4星：1小时只能承接1单
  - 2星 ≤ 评分 < 3星：5小时只能承接1单
  - 1星 ≤ 评分 < 2星：12小时只能承接1单
  - 评分 < 1星：24小时只能承接1单
• 承接任务时检查限制：
  - 帮助者点击"承接"时，检查是否在限制期内
  - 如果在限制期内，提示："您因评分过低，当前每X小时只能接1单，下一单可接时间为XX:XX"
• 默认评价：
  - 超过12小时自动完成的任务，帮助者默认5星评价
  - 举报后被客服判定有责的任务，帮助者默认1星评价

2.6 我的页面显示星级和评分

需求描述：
• 在"我的"页面，只有在帮助者设置中选择了"愿意帮助他人"的用户才显示星级和评分
• 显示内容包括：
  - 星级（如4.5星）
  - 具体分数（如4.53分）
  - 评价总数（如基于128次评价）
• 未选择帮助者身份的用户不显示

三、用户协议更新

在用户协议中增加以下内容：

【接单限制规则】

平台对帮助者的服务质量进行评价，并根据评价结果对帮助者的接单频率进行限制：

1. 评分≥4星（不含4星）：无接单限制
2. 3星≤评分＜4星：每小时最多承接1单
3. 2星≤评分＜3星：每5小时最多承接1单
4. 1星≤评分＜2星：每12小时最多承接1单
5. 评分＜1星：每24小时最多承接1单

评分计算方式：所有任务评价的平均分（四舍五入）。

系统会自动根据您的评分计算接单限制，承接任务时会进行校验。如需查看具体限制，请在"我的"页面查看您的当前评分和下次可接单时间。

【评价规则】

1. 任务完成后，求助者需对帮助者进行评价（1-5星）
2. 超过12小时自动完成的任务，帮助者默认获得5星评价
3. 举报成立后，帮助者默认获得1星评价
4. 评价提交后无法修改，请客观公正评价

【举报规则】

1. 求助者可以举报帮助者的以下行为：
   - 虚假信息：提供的信息与实际情况严重不符
   - 态度恶劣：使用侮辱性语言、威胁或骚扰
   - 违规交易：要求私下转账或交易
   - 其他：其他违反平台规则的行为

2. 举报提交后，系统将自动结束任务，积分暂时冻结

3. 客服将在48小时内处理举报，根据聊天记录和证据判定积分归属：
   - 举报成立：积分退还给求助者，帮助者获得1星差评
   - 举报不成立：积分转给帮助者
   - 情节严重者，平台有权封禁账号

4. 举报时需提供真实证据，恶意举报将影响您的信用评分

五、开发优先级

P0（必须完成）：
1. 任务超时12小时自动完成
2. 我的求助列表左滑删除
3. 接单限制逻辑
4. 消息中心消息置顶和删除
5. 我的页面显示星级
6. 举报功能（前台+后台）

P1（重要）：
1. 接单限制提示文案优化

P2（优化）：
1. 举报记录查询

六、开发顺序

1. 任务超时12小时自动完成 → 2. 我的求助左滑删除 → 3. 接单限制逻辑 → 4. 消息置顶/删除 → 5. 我的页显示星级 → 6. 举报前台+后台 → 7. 接单限制提示 → 8. 举报记录查询

七、风险与注意事项

• 接单限制规则需要明确告知用户，避免投诉
• 举报功能需要客服后台配合，建议先设计好数据模型
• 自动完成任务的时间检测需要服务端定时任务或前端轮询
• 假删除逻辑需要确保帮助者端不受影响
