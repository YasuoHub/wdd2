# 问当地产品设计说明

## 一、产品概述

### 1.1 产品定位
- **产品名称**：问当地
- **Slogan**：当地想问啥，就来问当地
- **核心定位**：基于地理位置的即时互帮互助小程序
- **目标**：解决用户"即时获取异地实时信息"的痛点

### 1.2 用户群体
- **双角色并行**：用户同时可以是求助者和帮助者
- **求助者**：需要获取异地实时信息的人（18-55岁）
- **帮助者**：愿意通过提供信息获取积分的人（18-60岁）

---

## 二、核心业务流程

### 2.1 求助者流程
```
① 打开小程序 → 微信授权登录（获得100积分）
② 点击发布求助
③ 填写：求助地点、求助类型、问题描述、任务有效期、悬赏积分
④ 发布成功 → 系统冻结积分
⑤ 等待帮助者接单
⑥ 匹配成功 → 进入聊天
⑦ 收到帮助者反馈（文字/图片）
⑧ 确认完成 → 评价
⑨ 积分解冻 → 积分转给帮助者 → 任务完成
```

### 2.2 帮助者流程
```
① 打开小程序 → 微信授权登录
② 完善个人信息（帮助意愿、常活动地点、可帮助类型、联系电话）
③ 收到微信服务通知 → 点击进入
④ 查看任务列表 → 选择承接任务
⑤ 进入聊天 → 提供信息反馈
⑥ 等待求助者确认完成
⑦ 积分到账
```

### 2.3 分支情况
- **积分不足**：无法发布，提示获取积分
- **超时取消**：24小时无人接单，自动取消，积分退还
- **客服介入**：待验收/任务执行中可发起

## 三、功能模块设计

### 3.1 积分体系

| 积分获取 | 数量 |
|----------|------|
| 新用户首次登录 | 100积分 |
| 邀请好友 | 双方各50积分（需完成首次求助） |
| 完成帮助任务 | 获得悬赏积分 |
| 连续签到 | 第1天5积分 → 第7天30积分 → 第8天重置 |

| 积分规则 | 说明 |
|----------|------|
| 总积分 | 可用积分 + 冻结积分 |
| 积分冻结 | 发布任务时冻结，完成/取消后解冻 |
| 积分有效期 | 当个自然年，次年1月1日清零 |
| 积分不可提现 | 仅限平台内消费（后续考虑开放） |

### 3.2 求助类型（6种）
- 实时天气
- 道路拥堵
- 店铺营业
- 停车场空位
- 排队情况
- 其他

### 3.3 任务有效期
- 最短30分钟，最长24小时
- 超时自动取消，积分自动解冻

### 3.4 任务接单规则
- 一人接单后，其他人不能再接
- 帮助者接单后需完成任务，否则影响信用

## 四、页面结构设计

### 4.1 页面清单（6个页面）

| 页面 | 功能 | 说明 |
|------|------|------|
| 首页 | 发布入口 + 附近任务预览 | Tab1 |
| 发布页 | 发布求助 | 6种类型+积分冻结 |
| 任务大厅 | 可接任务列表 | Tab2 |
| 聊天页 | 实时沟通 | 文字+图片 |
| 消息中心 | 统一消息入口 | Tab3 |
| 我的 | 积分+任务入口 | Tab4 |

### 4.2 身份标识系统

| 身份 | 图标 | 说明 |
|------|------|------|
| 求助者 | 🙏（双手抱拳） | 发出任务 |
| 帮助者 | 🤗（拥抱姿势） | 接收任务 |

- 使用图标区分身份，不使用颜色
- 每个页面顶部显示当前身份

### 4.3 页面关系

```
首页
├── 发布求助 → 发布页 → 聊天页
├── 附近任务 → 任务详情 → 聊天页
└── Tab切换 → 任务大厅 / 消息中心 / 我的

任务大厅 → 任务详情 → 聊天页

消息中心 → 聊天页

我的
├── 我的求助列表 → 任务详情/聊天页
└── 已接任务列表 → 任务详情/聊天页
```

## 五、关键交互说明

### 5.1 任务状态与操作

| 任务状态 | 身份 | 操作 |
|----------|------|------|
| 进行中 | 求助者 | 进入聊天、确认完成 |
| 进行中 | 帮助者 | 进入聊天、提供信息 |
| 已完成 | 双方 | 查看记录（不可聊天） |
| 已取消 | 双方 | 查看记录（不可聊天） |

### 5.2 聊天功能
- 支持发送文字
- 支持发送图片（拍照/相册）
- 消息实时显示

### 5.3 消息中心
- 统一展示所有任务相关通知
- 区分求助/帮助角色
- 可点击进入对应任务或聊天

## 六、技术方案

### 6.1 技术栈选择
- **小程序**：微信原生开发
- **后端**：微信云开发（云函数+云数据库）
- **存储**：云存储（图片）
- **地图**：腾讯地图SDK

### 6.2 云函数命名规范
- 前缀：`wdd-`
- 例如：`wdd-login`, `wdd-publish`, `wdd-match`

### 6.3 数据库集合（待创建）
- users（用户）
- needs（任务）
- messages（聊天记录）
- point_records（积分记录）
- sign_in_records（签到记录）
- notifications（通知）

## 九、产品特色

1. **双角色并行**：用户同时是求助者和帮助者，降低使用门槛
2. **积分激励**：通过积分体系驱动用户参与
3. **地理位置匹配**：精准匹配附近帮助者
4. **即时沟通**：文字+图片反馈，信息更真实
5. **轻量互助**：不同于美团骑手，是碎片化的轻量互助
