## 发现问题清单（共7项）

### 问题1: wdd-take-need 事务违规
**位置**: `cloudfunctions/wdd-take-need/index.js` 第80行
**问题**: 事务内直接 `update` wdd-needs，未先执行 `get`。
**云开发事务规则**: 同一文档在事务内必须先 `get` 才能 `update`。
**修复方案**: 在事务开始后立即添加 `await transaction.collection('wdd-needs').doc(needId).get()`，并用事务内查询结果替换后续 update 中的外部变量。

### 问题2: wdd-sign-in 事务违规
**位置**: `cloudfunctions/wdd-sign-in/index.js` 第139行
**问题**: 事务内直接 `update` wdd-users，未先执行 `get`。
**修复方案**: 在事务开始后立即添加 `await transaction.collection('wdd-users').doc(userId).get()`，用事务内查询的最新 `total_points` 计算流水 balance。

### 问题3: wdd-publish 事务违规
**位置**: `cloudfunctions/wdd-publish/index.js` 第110行
**问题**: 事务内直接 `update` wdd-users，未先执行 `get`。
**修复方案**: 在事务开始后立即添加 `await transaction.collection('wdd-users').doc(userId).get()`，用事务内查询的最新 `total_points` 计算流水 balance。

### 问题4: wdd-login 事务违规
**位置**: `cloudfunctions/wdd-login/index.js` 第110行
**问题**: 事务内直接 `update` wdd-users（inviter），未先执行 `get`。
**修复方案**: 在事务开始后立即添加 `await transaction.collection('wdd-users').doc(inviter._id).get()`，用事务内查询的最新积分计算流水 balance。

### 问题5: wdd-auto-cancel 事务违规
**位置**: `cloudfunctions/wdd-auto-cancel/index.js` 第80行
**问题**: 事务内直接 `update` wdd-needs，未先执行 `get`。
**修复方案**: 在事务开始后立即添加 `await transaction.collection('wdd-needs').doc(need._id).get()`，并用事务内查询结果进行后续操作。

### 问题6: wdd-payment/confirmPayment 变量作用域问题
**位置**: `cloudfunctions/wdd-payment/index.js` 第181行附近
**问题**: 事务内使用了**事务外**查询的 `metadata` 和 `order.amount` 变量。虽然本业务场景下这些值几乎不会被并发修改，但严格违反事务一致性原则。
**修复方案**: 将 `const metadata = orderRes.data.metadata || {}` 移入事务内，改为 `const metadata = orderInTx.metadata || {}`；所有使用 `order.amount` 的地方改为 `orderInTx.amount`。

### 问题7: wdd-payment/refundOrder 用户查询使用错误 openid
**位置**: `cloudfunctions/wdd-payment/index.js` 第417-420行
**问题**: 事务内查询用户时使用了 `OPENID`，但云函数间调用时 `OPENID` 为空，正确值应使用 `callerOpenid`（`event.openid || OPENID`）。
**修复方案**: 将 `openid: OPENID` 改为 `openid: callerOpenid`。

### 步骤2: 端到端测试验证

#### 测试1: 支付 → 发布 → 查看 → 取消 → 退款
1. 发布一个求助任务，悬赏 0.01 元
2. 确认调起真实微信支付弹窗
3. 支付完成后确认任务创建成功
4. 进入"我的求助"列表，点击该任务
5. 点击"取消任务"
6. 确认收到"任务已取消，悬赏金额已原路退回"提示

#### 测试2: 支付 → 发布 → 接单 → 完成 → 结算
1. 发布一个求助任务
2. 用另一个账号接单
3. 求助者确认完成
4. 帮助者检查余额是否增加（扣除平台抽成后）
5. 帮助者检查余额流水记录是否正确

#### 测试3: 自动取消
1. 发布一个求助任务（有效期设最短）
2. 等待超时（或手动修改数据库过期时间测试）
3. 确认任务自动取消
4. 确认微信支付收到退款

#### 测试4: 提现
1. 帮助者完成一个任务获得余额
2. 进入钱包页面申请提现
3. 确认余额正确扣除，提现记录生成

## 风险提示

1. **企业付款到零钱**: 提现的真实打款需要商户入驻满90天+连续交易30天，当前保持人工审核模式。
2. **退款幂等性**: 微信退款使用固定 `outRefundNo = 'RF' + out_trade_no`，同一订单重复退款不会重复打款。
3. **积分体系**: `wdd-publish`、`wdd-sign-in`、`wdd-login` 仍使用积分字段，与真实支付并存。积分仅用于签到/邀请奖励，任务发布使用微信支付。
