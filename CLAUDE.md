## 项目概述

**问当地** - 基于地理位置的即时互帮互助微信小程序。

- **Slogan**: 当地想问啥，就来问当地
- **核心定位**: 连接有异地实时信息需求的求助者与对应地点的帮助者
- **双角色并行**: 用户同时可以是求助者（🙏）和帮助者（🤗）

## 关键决策（第一阶段确定）

### 冷启动策略
- **首批运营城市**：成都
- **运营方式**：聚焦单一城市启动，确保匹配成功率
- **早期支持**：平台预设问答内容度过冷启动期

### 积分体系
- **MVP阶段**：明确告知用户"积分未来可提现"
- **激励机制**：新用户100积分，后续支持签到、邀请等方式获取

### 需求验证
- **方式**：直接构建MVP，边做边验证，根据反馈快速迭代

### 核心参数
| 参数 | 设定值 | 说明 |
|------|--------|------|
| 匹配距离 | 求助者选择，默认5公里 | 以直线距离计算 |
| 任务有效期 | 默认30分钟 | 主打即时信息获取，最长24小时 |
| 通知间隔 | 5分钟 | 帮助者不响应则通知下一位 |
| 积分有效期 | 当年有效 | 次年1月1日清零 |
| 积分兑换比例 | 10积分 = 1元 | 未来提现时的兑换比例 |

### 帮助者动力定位
- **核心逻辑**：不是"赚钱平台"，而是"顺手帮忙赚点小钱"
- **场景假设**：帮助者本来就要去那个地方，顺便看一眼帮个忙
- **价值交换**：举手之劳换一杯奶茶钱（如5积分=0.5元）

### 微信审核策略
- **产品定位**：强调"社区互助"，避免"交易平台"属性
- **内容安全**：接入微信内容安全接口（msgSecCheck）审核聊天内容
- **资金说明**：MVP阶段仅积分流转，暂不支持提现

## 关键业务规则

### 积分体系
- 新用户首次登录：100 积分
- 总积分 = 可用积分 + 冻结积分
- 发布任务时冻结积分，完成/取消后解冻

### 流程测试（端到端场景）

#### 场景1：完整成功流程
```
用户A（求助者）            用户B（帮助者）
    |                          |
    | 1. 发布任务（10积分）       |
    |-------------------------->|
    |                          | 2. 任务大厅看到任务
    |                          | 3. 点击接单
    |<--------------------------|
    | 4. 进入聊天               |
    |<-----> 5. 文字/图片沟通 --->|
    |                          |
    | 6. 确认完成               |
    |-------------------------->|
    |                          | 7. 收到10积分
    | 8. 评价5星                |
```
**验证点**：
- 用户A：冻结10 → 解冻0（完成），总积分-10
- 用户B：积分+10
- 任务状态：待匹配 → 进行中 → 已完成

#### 场景2：超时取消流程
```
用户A 发布任务（有效期30分钟）
      ↓
   无人接单
      ↓
   30分钟后
      ↓
   任务自动取消
      ↓
   积分退还用户A
```
**验证点**：
- 任务状态变为"已取消"
- 用户A冻结积分10 → 解冻回到可用积分
- 用户A收到系统通知"任务超时已取消"

#### 场景3：积分不足场景
```
用户A 现有积分20
      ↓
   发布任务悬赏30积分
      ↓
   提示"积分不足"
      ↓
   引导去签到获取积分
      ↓
   签到获得5积分（25分）
      ↓
   仍不足，继续签到或降低悬赏
```

### 边界情况测试

| 场景 | 测试内容 | 预期处理 |
|------|---------|---------|
| 弱网环境 | 网络不稳定时发消息 | 显示发送中 → 发送失败/重试 |
| 同时多任务 | 一个用户同时发布多个求助 | 允许，分别冻结积分 |
| 位置权限拒绝 | 用户不允许获取位置 | 提示手动输入地址 |
| 空内容发送 | 聊天时发送空消息 | 按钮置灰，禁止发送 |
| 超长内容 | 问题描述输入1000字 | 限制最多500字，超出截断或提示 |
| 图片过大 | 发送10MB图片 | 压缩或提示"图片过大" |
| 并发抢单 | 两人同时点击接单 | 一人成功，一人提示"已被接单" |
