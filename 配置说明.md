## 平台全局配置（wdd-config）

`wdd-config` 集合用于存储平台运行时配置，支持**在线修改无需重新部署代码**。

### 配置文档

集合中需创建一条 `_id: 'platform'` 的文档：

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `platform_fee_rate` | number | 0.15 | 平台服务费抽成比例（如 0.15 = 15%） |
| `withdraw_fee_rate` | number | 0.01 | 提现手续费率（如 0.01 = 1%），按 `wdd-config` 动态配置 |
| `withdraw_min_amount` | number | 0.01 | 旧最低提现门槛字段，当前版本不再作为用户提现门槛 |
| `withdraw_min_per_request` | number | 1 | 微信商家转账单笔最低金额，需与商户平台产品设置一致 |
| `withdraw_max_per_request` | number | 5000 | 单次提现最高金额（元） |
| `withdraw_approval_threshold` | number | 100 | 旧提现审批阈值字段，当前版本停用人工提现审批 |
| `withdraw_daily_limit` | number | 100 | 单日提现累计金额上限（元） |
| `withdraw_daily_times` | number | 1 | 单日提现次数上限（次） |
| `min_reward_amount` | number | 1 | 最小悬赏金额（元） |
| `max_reward_amount` | number | 500 | 最大悬赏金额（元） |
| `register_gift_balance` | number | 3 | 新用户注册赠送现金余额（元） |
| `points.register` | number | 100 | 新用户注册奖励积分 |
| `points.invite` | number | 50 | 邀请奖励积分 |
| `points.signIn.daily` | number[] | [5,10,15,20,25,30,30] | 连续签到每日奖励积分 |
| `customer_service_openids` | string[] | [] | 客服 OpenID 白名单 |
| `max_transfer_retry` | number | 5 | 提现转账失败后的最大重试次数 |
| `transfer_backoff_minutes` | number[] | [5,10,20,40,80] | 提现转账重试间隔（分钟） |
| `transfer_query_timeout_minutes` | number | 1 | 提现转账查询超时分钟数 |

### 初始化配置

在微信开发者工具的云数据库中，创建 `wdd-config` 集合并插入文档：

```json
{
  "_id": "platform",
  "platform_fee_rate": 0.15,
  "withdraw_fee_rate": 0.01,
  "withdraw_min_amount": 0.01,
  "withdraw_min_per_request": 1,
  "withdraw_max_per_request": 5000,
  "withdraw_approval_threshold": 100,
  "withdraw_daily_limit": 100,
  "withdraw_daily_times": 1,
  "min_reward_amount": 1,
  "max_reward_amount": 500,
  "register_gift_balance": 3,
  "points": {
    "register": 100,
    "invite": 50,
    "signIn": {
      "daily": [5, 10, 15, 20, 25, 30, 30]
    }
  },
  "customer_service_openids": [],
  "max_transfer_retry": 5,
  "transfer_backoff_minutes": [5, 10, 20, 40, 80],
  "transfer_query_timeout_minutes": 1
}
```

### 修改配置

超级管理员可在小程序「我的 - 超级管理员 - 系统配置」页面维护上述配置。也可以直接在云数据库中编辑 `wdd-config/platform` 文档，修改后**无需修改代码、无需重新部署云函数**。

> `super_admin_openids` 属于超级管理员身份配置，不在系统配置页面中开放修改，避免误删或误授权。

配置变更会在以下时机自动加载：
- 小程序启动时（`app.onLaunch`）
- 小程序从后台切回前台时（`app.onShow`）

### 受影响的页面和功能

| 功能 | 受影响的配置 |
|------|-------------|
| 发布任务页（金额计算展示） | `platform_fee_rate` |
| 任务大厅接单弹窗 | `platform_fee_rate` |
| 任务详情页（金额计算） | `platform_fee_rate` |
| 任务结算（帮助者实际到账） | `platform_fee_rate` |
| 钱包页（提现规则提示） | `withdraw_daily_limit`、`withdraw_daily_times` |
| 提现页（金额校验） | `withdraw_max_per_request`、`withdraw_daily_limit`、`withdraw_daily_times` |
| 历史提现审批（当前停用） | `withdraw_approval_threshold` 仅作旧字段保留 |
| 发布页（悬赏金额范围） | `min_reward_amount`、`max_reward_amount` |
| 新用户注册现金余额奖励 | `register_gift_balance` |
| 登录注册积分奖励 | `points.register`、`points.invite` |
| 每日签到积分 | `points.signIn.daily` |
| 客服入口和工单处理权限 | `customer_service_openids` |
| 提现转账重试策略 | `max_transfer_retry`、`transfer_backoff_minutes`、`transfer_query_timeout_minutes` |

### 注意事项

- 若数据库中不存在配置文档，前后端均回退到代码中的硬编码默认值
- 云函数 `wdd-get-config` 负责读取配置并返回给前端，**需部署后前端才能获取配置**
- 结算云函数 `wdd-settlement` 直接在结算时从数据库读取 `platform_fee_rate`，与前端展示独立
