# 问当地 UI 设计图生成提示词合集：城市现场情报风

**用途**：这是“问当地”的第二套视觉方向，用于让图像生成 AI 直接生成微信小程序页面设计图。  
**适用对象**：首页、发布求助、任务大厅、任务详情、聊天、消息、我的、钱包、提现、评价、举报、申诉、客服工单、运营分析、资金审批等现有页面。  
**与旧版区别**：旧版是“青春活力、天蓝亲和”。本版改为“城市现场情报 + 可信交易 + 低调专业”，更适合公开发布后的真实服务感。

## 0. 使用方法

1. 每次生成一个页面，先复制“全局提示词”，再复制对应“页面提示词”。
2. 页面按微信小程序移动端竖屏设计，宽度按 750rpx 理解，画面像真实 App 截图，不要像海报。
3. 生成页面后，图标、顶部纹理、空状态插画、地图占位图等需要复用的视觉资产要单独生成 PNG。
4. 如果文字不准，只要求 AI 重画当前页面，不要一次改多个页面。

## 1. 全局提示词

```text
设计一个微信小程序移动端 UI 页面，产品名为“问当地”。

产品定位：
“问当地”是一个基于地理位置的实时信息互助小程序。用户可以发布带悬赏的本地求助，例如确认某地是否下雨、某条路是否拥堵、某家店是否营业、停车场是否有空位、现场排队是否严重；附近帮助者可以在任务大厅接单，通过聊天提供现场信息并获得收益。

当前产品功能结构：
底部 Tab 为：首页、大厅、消息、我的。页面包括发布求助、任务详情、聊天、我的求助、我的接单、钱包、提现、评价、帮助者资料、公开资料、举报、申诉、客服工单、运营分析、资金审批。产品已经包含真实交易闭环、微信支付、余额支付、冻结金额、提现审批、举报申诉和客服裁定，因此界面必须可信、清楚、专业。

整体视觉方向：
采用“城市现场情报风”。界面像一张干净的城市信息工作台，而不是社交 App、外卖抢单平台或金融理财产品。关键词：现场感、地点感、低调专业、可信交易、即时反馈、轻量人情味。

视觉气质：
清醒、利落、可靠，有一点地图、路牌、收据、任务单、现场照片的感觉。不要青春糖果风，不要大面积天蓝渐变，不要可爱卡通，不要炫酷科技风，不要厚重金融风，不要营销落地页。

画布规格：
微信小程序手机竖屏 UI，按 750rpx 宽度设计。保留微信小程序顶部导航栏和底部 Tab 区域。首屏必须像真实产品截图，元素完整、文字可读、信息层级清楚。

色彩规范：
页面底色使用“雾白灰” #F6F5F1 或 #F4F6F3。
卡片使用“纸白” #FFFFFF。
主文字使用“石墨黑” #1F2933。
次文字使用“灰绿” #66746B。
弱文字使用“冷灰” #96A19A。
主色使用“坐标墨绿” #1F6F5B，用于主按钮、选中态、重要图标和当前位置感。
强调色使用“悬赏琥珀” #D9822B，用于金额、支付、重要收益。
辅助蓝使用“路牌蓝” #2F80A8，用于链接、消息、信息提示。
成功色使用“完成绿” #3D9A6A，用于完成、收益到账、已通过。
提醒色使用“倒计时黄” #E3B341，用于等待、即将超时、积分。
风险色使用“证据红” #C94C4C，用于举报、驳回、错误，但只做局部提示，不要大面积铺红。
边框使用 #E4E7E2，分割线使用 #EEF0EC。

纹理与背景：
页面背景可以有极淡的地图等高线、坐标网格、票据压纹或城市路网线条，透明度很低，只作为气氛。不要使用渐变球、光斑、霓虹、玻璃拟态、大幅插画背景。顶部区域可以有一条细的“定位坐标/任务单编号”式装饰线。

圆角与阴影：
整体比旧版更克制。卡片圆角 12-16rpx，按钮圆角 12-20rpx，标签圆角 8-12rpx。不要超大胶囊和软萌圆角。阴影极轻，优先用边框和留白分层，阴影最多为 0 6rpx 18rpx rgba(31, 41, 51, 0.06)。

排版规范：
使用清晰现代的中文无衬线字体，偏向“工作台/信息牌”的阅读感。页面标题 36-40rpx，模块标题 30-32rpx，正文 27-28rpx，辅助文字 22-24rpx。金额数字可用更紧凑的数字风格，明显但不夸张。文字不要负字距，不要用超大标题压过功能。

组件风格：
任务卡片像一张“现场求助单”，必须突出：类型、地点、距离、悬赏、剩余时间、问题摘要、状态。左侧可用窄色条或小坐标点表示类型，不要用大插画。
表单页像“填写任务单”，分组明确，字段之间留白清楚。地点、类型、图片、金额、支付方式都要能一眼区分。
聊天页像“围绕任务的现场沟通”，顶部固定任务摘要，消息气泡克制，照片缩略图清楚。
钱包、提现、资金审批页面像“交易凭证/账本”，金额、冻结、审批状态、手续费必须清楚，不要理财感。
举报、申诉、客服工单页面像“提交事实材料”，强调证据、流程、处理状态，避免情绪化警报风。
运营分析页面像“移动端运营仪表盘”，数字和图表清楚，信息密度高但不拥挤。

图标规范：
不要使用 emoji。使用统一 2px 线性图标，圆角端点，风格像城市路牌和地图工具。图标语义包括：定位、天气、道路、店铺、停车、排队、金额、时间、聊天、图片、钱包、提现、评价、举报、申诉、工单、运营图表、审批。图标颜色以墨绿、琥珀、路牌蓝、完成绿、风险红为主，透明背景。

交互视觉：
所有按钮必须有明确主次。底部固定操作栏避开安全区。列表适合快速扫读，表单适合逐步填写，弹窗适合确认资金或争议操作。每个页面只突出一个主要目标。不要在页面中写产品理念，不要堆长说明。

输出要求：
生成真实可发布的小程序 UI 效果图，不要线框图，不要概念海报。页面里的中文尽量准确，按钮文案短，状态文案短。明显需要切图复用的图标、空状态图、地图占位图、顶部纹理单独输出 PNG，透明背景，无白底，无多余边距。
```

## 2. 关键组件设计规范

### 2.1 任务卡片

```text
任务卡片设计为“现场求助单”。白色纸面卡片，浅灰边框，左侧 6rpx 窄色条表示任务类型。顶部左侧是类型图标和类型名称，右侧是悬赏金额。中部是 2 行以内的问题摘要。底部第一行显示地点和距离，第二行显示发布时间、剩余时间和状态。接单按钮放在右下角，使用墨绿色实心按钮。金额使用琥珀色，倒计时使用黄褐色，风险状态使用红色小标签。
```

### 2.2 地点模块

```text
地点模块不要做复杂真实地图。使用浅灰绿色地图占位块，里面有淡淡路网线条、坐标点、地点名称和地址。右侧可有“查看地图”箭头。地点信息要像导航 App 的地点卡，但更轻量。
```

### 2.3 资金模块

```text
资金模块像一张清楚的交易凭证。金额大但克制，背景白色或极浅灰，不使用大面积金色渐变。冻结金额、可用余额、手续费、预计到账分行展示。关键金额使用琥珀色或石墨黑，收入使用完成绿，支出使用风险红，冻结使用倒计时黄。
```

### 2.4 争议处理模块

```text
举报、申诉、工单页面使用“事实材料”风格。表单项清楚，证据图片区明显，状态标签克制。红色只用于风险标记和错误状态，不要整页红色。确认弹窗要表达“提交后进入客服评审、聊天通道可能关闭、平台依据证据处理”。
```

### 2.5 空状态

```text
空状态使用极简线性插画：地图坐标点、空任务单、空收据、空消息气泡。插画颜色低饱和，面积不超过页面首屏的 25%。空状态文案只写一句原因和一个下一步按钮。
```

## 3. 页面提示词

### 3.1 首页 pages/index/index

```text
基于“城市现场情报风”，设计微信小程序【首页】。

页面目标：
让用户立刻知道：可以发布一个异地实时求助，也可以去大厅接单帮助别人。首页是实际产品入口，不是营销页。

布局：
顶部保留微信小程序导航栏，标题“问当地”。内容顶部是简洁品牌区，左侧显示“问当地”，下方小字“城市现场信息互助”。右侧显示一个小坐标徽标，不要大插画。

核心入口：
首屏最重要模块是“发布求助”任务单卡片。卡片像一张待填写的城市情报单，标题“发布求助”，副文案“想知道某地现场情况，发给当地人确认”。卡片下方放 3 个小标签：“选地点”“设悬赏”“等回复”。右侧有墨绿色主按钮“去发布”。

附近求助：
模块标题“附近求助”，旁边显示“5公里内”。展示 2 张任务卡片：店铺营业 ¥8，春熙路 · 1.2km；停车空位 ¥12，太古里 · 3.6km。卡片要突出地点、金额和剩余时间。

未登录状态：
显示一张克制登录卡，文案“登录后查看附近求助”，按钮“微信登录”。不要使用大卡通插画。

底部 Tab：
首页、大厅、消息、我的。当前选中首页，选中图标和文字为墨绿色。
```

### 3.2 发布求助页 pages/publish/publish

```text
基于“城市现场情报风”，设计微信小程序【发布求助页】。

页面目标：
帮助用户快速发布一笔真实悬赏求助。页面像填写一张清楚的任务单，要简单、可信、金额明确。

布局：
顶部导航标题“发布求助”。页面主体为纵向表单，每个分组是白色纸面卡片，圆角 12-16rpx，边框浅灰。

分组顺序：
1. 求助地点：地点选择卡，未选中显示“请选择地点”“在地图上选择你想了解的位置”；已选中显示地点名和地址。背景可用浅路网纹理。
2. 求助类型：六个线性图标选择项，实时天气、道路拥堵、店铺营业、停车空位、排队情况、其他。选中项用墨绿色边框和浅绿底。
3. 详细描述：多行输入框，占位示例“例如：这家店现在还营业吗？门口排队大概多久？”
4. 添加图片：最多 3 张，上传位像现场凭证格。
5. 有效时间：30分钟、1小时、2小时、4小时、12小时、24小时 chips。
6. 悬赏金额：金额输入和快捷金额，金额用琥珀色；下方展示服务费和帮助者实际获得。
7. 支付方式：余额支付、微信支付两张横向卡。余额不足时用小琥珀提示。

底部固定操作栏：
左侧显示“需支付 ¥8.00”，右侧主按钮“支付并发布”。底部栏白底、上边框、避开安全区。

视觉重点：
金额、支付、地点三件事必须一眼清楚。不要使用大面积渐变和可爱装饰。
```

### 3.3 任务大厅页 pages/task-hall/task-hall

```text
基于“城市现场情报风”，设计微信小程序【任务大厅页】。

页面目标：
帮助者快速扫读附近可接任务，判断是否值得接单。

布局：
顶部导航标题“任务大厅”。页面顶部直接是横向类型筛选：全部、天气、道路、店铺、停车、排队、其他。筛选项像小路牌标签，选中为墨绿色。

次级筛选：
类型筛选下方放两个小按钮：“距离最近”和“5公里内”，点击后是底部弹窗。弹窗选项有对勾。

帮助者资料提示：
如果未完善资料，显示浅绿提示条“完善帮助者资料”，按钮“去完善”，可关闭。提示条不要抢走列表焦点。

任务列表：
列表标题“附近共有 18 个求助”。任务卡片采用现场求助单样式。每张卡片包含类型、悬赏、问题摘要、地点、距离、剩余时间、发布人头像和“接单”按钮。

示例任务：
店铺营业 ¥8，春熙路咖啡店现在还营业吗，春熙路 · 1.2km，剩余 47 分钟。
道路拥堵 ¥15，二环高架往成渝立交方向现在堵不堵，二环高架 · 2.8km。
停车空位 ¥12，太古里地下停车场现在还有空位吗，太古里 · 3.6km。

空状态：
显示空任务单线性插画，文案“附近暂无求助任务”，按钮“查看全部类型”。
```

### 3.4 任务详情页 pages/task-detail/task-detail

```text
基于“城市现场情报风”，设计微信小程序【任务详情页】。

页面目标：
让用户清楚了解任务内容、地点、悬赏、状态、双方角色和下一步操作。

布局：
顶部导航标题“任务详情”。顶部状态卡像一张任务单抬头，左侧状态“待匹配/进行中/审核中/已完成”，右侧大号悬赏“¥8”。

任务信息：
卡片内显示类型标签、问题摘要、发布时间。求助内容单独成区，文字可读，最多两三行示例。

地点信息：
地点卡使用浅路网占位，不放复杂真实地图。显示“求助位置”、地点名、地址或距离，右侧“查看地图”。

双方信息：
展示求助者、帮助者。未接单时帮助者区域显示“等待接单”。已接单时显示头像、昵称和“查看资料”入口。

时间信息：
发布时间、接单时间、完成时间，有值才展示。

底部操作栏：
待匹配帮助者视角显示主按钮“立即接单 +¥7.20”。进行中显示“进入聊天”“确认完成”。审核中显示状态提示，不显示主操作。

视觉重点：
详情页比列表页更稳重，像一张可追踪的任务凭证。
```

### 3.5 聊天页 pages/chat/chat

```text
基于“城市现场情报风”，设计微信小程序【聊天页】。

页面目标：
这是围绕某一笔任务的现场沟通，不是普通社交聊天。用户必须始终知道任务、地点、悬赏和状态。

布局：
顶部导航标题“聊天”。导航下固定任务摘要条，显示类型、地点、问题摘要、悬赏金额和更多菜单按钮。

消息区：
背景为雾白灰。对方消息左侧白色气泡，自己消息右侧浅墨绿气泡。气泡圆角克制，文字清楚。图片消息显示现场照片缩略图，带轻边框。

示例消息：
帮助者：“我到门口了，现在还在营业。”
帮助者发送一张门口照片。
求助者：“谢谢，门口排队大概多久？”
帮助者：“现在大概 3 个人，预计 5 分钟。”

更多菜单：
菜单项为“完成”“举报”“详情”，线性图标，不用 emoji。

底部输入栏：
白底固定，左侧附件按钮，中间输入框“输入消息”，右侧发送按钮。工具区包含“相册”“拍照”。

特殊状态：
审核中时底部显示“审核中，暂不可发送消息”。任务结束时显示“任务已结束”。
```

### 3.6 消息页 pages/messages/messages

```text
基于“城市现场情报风”，设计微信小程序【消息中心】。

页面目标：
让用户快速查看任务会话和系统通知，知道哪些任务有新进展。

布局：
顶部导航标题“消息中心”。顶部是分段切换“求助”“帮助”，右上角有系统通知铃铛和未读角标。

会话列表：
每条会话像一条任务通信记录，左侧头像或类型图标，中间显示任务类型、地点、问题摘要、最后一条消息，右侧显示时间和未读角标。进行中任务有墨绿色状态点，审核中有琥珀状态点。

视觉重点：
消息页不是普通聊天列表，要突出“这是哪一单任务的消息”。
```

### 3.7 我的页 pages/my/my

```text
基于“城市现场情报风”，设计微信小程序【我的】页面。

页面目标：
汇总用户身份、余额积分、任务入口、帮助者资料、服务记录和管理入口。

布局：
顶部用户卡不做大渐变，使用白色纸面卡。左侧头像和昵称，右侧小标签显示“求助者/帮助者”。下方横向展示余额和积分，点击进入钱包或积分记录。

签到：
签到按钮像一张小票据按钮，未签到为墨绿色，已签到为浅灰绿色。

菜单：
分组为“我的任务”“帮助者设置”“资产管理”“服务记录”“管理入口”“其他”。菜单项使用统一线性图标，右侧箭头。不要使用 emoji。

管理员入口：
客服工单、运营分析、资金审批如果有权限展示，样式更克制，使用工作台风格图标。
```

### 3.8 钱包页 pages/wallet/wallet

```text
基于“城市现场情报风”，设计微信小程序【钱包】页面。

页面目标：
让用户清楚理解余额、冻结金额、可提现金额、提现申请和收支明细。
```

## 4. 独立视觉资产提示词

### 4.1 图标资产

```text
为微信小程序“问当地”生成一组统一线性图标，风格为城市现场情报风。每个图标单独输出 80x80px 2倍高清 PNG，透明背景，无白底，无阴影外溢，无多余边距。线宽统一，圆角端点，颜色使用坐标墨绿 #1F6F5B、悬赏琥珀 #D9822B、路牌蓝 #2F80A8、完成绿 #3D9A6A、风险红 #C94C4C。图标包括：实时天气、道路拥堵、店铺营业、停车空位、排队情况、其他、定位、时间、悬赏、聊天、图片、钱包、提现、评价、举报、申诉、工单、运营分析、资金审批。不要使用 emoji、卡通、复杂渐变或照片质感。
```


## 5. 设计判断标准

1. 第一眼能看出这是“实时地点信息互助”，不是普通社交、外卖抢单或金融 App。
2. 发布、接单、聊天、结算这条主链路清楚。
3. 金额、冻结、提现、审批、举报、申诉等高风险信息清楚可信。
4. 页面足够专业，可以公开发布给真实用户。
5. 与现有代码结构兼容：底部 Tab、页面顺序、任务状态、支付方式、举报申诉、工单和运营页面不虚构新功能。
