## 一、产品概述

### 1.1 产品简介

"问当地"是一款基于地理位置的即时互帮互助微信小程序。产品以"地理位置匹配"为核心，连接有异地实时信息需求的求助者与对应地点的帮助者，解决用户"即时获取异地实时信息"的痛点。

**产品口号**: 当地想问啥，就来问当地

### 1.2 核心定位

- **双角色并行**: 用户同时可以是求助者（🙏）和帮助者（🤗）
- **地理位置匹配**: 基于用户实时位置进行智能匹配
- **即时信息获取**: 主打实时性，解决用户无法亲临现场的信息需求

### 1.3 应用场景

| 场景类型 | 具体需求 | 示例 |
|---------|---------|------|
| 出差前查询 | 实时天气 | 明天去北京出差，现在北京下雨吗？ |
| 自驾出行 | 道路拥堵 | 去机场的路上堵不堵？ |
| 购物决策 | 店铺营业 | 这家店现在还开着吗？ |
| 停车规划 | 停车场空位 | 目的地附近好停车吗？ |
| 排队预判 | 排队情况 | 医院现在人多吗？ |

## 二、功能规格

### 2.1 用户系统

#### 2.1.1 用户注册与登录
- **登录方式**: 微信授权一键登录
- **用户信息**: 自动获取微信头像、昵称
- **新用户福利**: 首次登录赠送100积分
- **身份标识**: 🙏求助者 / 🤗帮助者

#### 2.1.2 用户资料管理
- 头像、昵称编辑
- 积分余额查看（可用积分 + 冻结积分）
- 帮助者资料设置（常去地点、帮助意愿、帮助类型）

### 2.2 求助功能

#### 2.2.1 发布求助
- **求助类型**（6种）:
  1. 实时天气
  2. 道路拥堵
  3. 店铺营业
  4. 停车场空位
  5. 排队情况
  6. 其他

- **必填信息**:
  - 求助类型
  - 求助地点（地图选点，支持搜索）
  - 问题描述（最多500字）
  - 悬赏金额（现金，单位元，最低 1 元）
  - 有效期（30分钟/1小时/2小时/4小时/12小时/24小时）

- **图片上传**: 最多3张图片，支持拍照和相册选择

#### 2.2.2 资金流转机制
- 发布时通过微信支付完成扣款（cloud.cloudPay.unifiedOrder）
- 任务完成后扣除平台抽成，剩余金额结算至帮助者钱包
- 任务取消/超时时通过微信原路退款给求助者

### 2.3 帮助功能

#### 2.3.1 任务匹配
- **匹配方式**: 基于帮助者实时位置计算距离
- **筛选条件**:
  - 距离范围（1/3/5/10公里/全部）
  - 求助类型
  - 排序方式（距离最近/积分最高/时间最新）

#### 2.3.2 接单流程
- 浏览任务大厅
- 查看任务详情
- 一键接单（同一时间只能接一个任务）
- 接单后双方进入聊天

### 2.4 实时通讯

#### 2.4.1 聊天功能
- **消息类型**: 文字、图片
- **推送机制**:
  - 优先使用云开发 watch 实时推送
  - 降级方案：3秒轮询
- **消息状态**: 未读消息角标提醒

#### 2.4.2 任务状态流转
```
待匹配 (pending) → 进行中 (ongoing) → 已完成 (completed)
                    ↓
              已取消 (cancelled)
```

### 2.5 积分系统（权益代币）

#### 2.5.1 积分获取
- 新用户注册：+100积分
- 每日签到：+5~30积分（连续签到递增）
- 邀请好友：双方各+50积分

#### 2.5.2 积分用途
- 用于平台内权益兑换（免单券、提现免手续费券等）
- **MVP 阶段兑换功能不开放**，前端展示"开发中"文案

#### 2.5.3 积分规则
- **重要说明**：积分仅用于平台内权益兑换，**不可提现、不可转让、无现金价值**
- **有效期**: 当前版本不设有效期，平台保留调整规则的权利
- 平台保留调整积分发放规则、可兑换权益种类与比例的权利

### 2.6 评价系统
- 任务完成后双方可互相评价
- 星级评分（1-5星）
- 评价内容展示在任务详情

### 3.3 数据模型

#### 3.3.1 用户表（wdd-users）
```javascript
{
  _id: String,              // 用户ID
  openid: String,           // 微信openid
  nickname: String,         // 昵称
  avatar: String,           // 头像URL
  total_points: Number,     // 总积分
  available_points: Number, // 可用积分
  frozen_points: Number,    // 冻结积分
  help_willingness: String, // 帮助意愿
  frequent_locations: [{    // 常去地点
    name: String,
    latitude: Number,
    longitude: Number
  }],
  help_types: [String],     // 愿意帮助的类型
  create_time: Date,
  update_time: Date
}
```

#### 3.3.2 求助表（wdd-needs）
```javascript
{
  _id: String,              // 任务ID
  user_id: String,          // 求助者ID
  user_nickname: String,    // 求助者昵称
  user_avatar: String,      // 求助者头像
  location: {               // 位置（GeoJSON格式）
    type: "Point",
    coordinates: [longitude, latitude]
  },
  location_name: String,    // 地点名称
  type: String,             // 求助类型
  type_name: String,        // 类型名称
  description: String,      // 问题描述
  images: [String],         // 图片URL列表
  points: Number,           // 悬赏积分
  status: String,           // 状态：pending/ongoing/completed/cancelled
  expire_time: Date,        // 过期时间
  taker_id: String,         // 帮助者ID
  taker_nickname: String,   // 帮助者昵称
  create_time: Date,
  update_time: Date
}
```

#### 3.3.3 聊天记录表（wdd-messages）
```javascript
{
  _id: String,
  need_id: String,          // 关联的任务ID
  sender_id: String,        // 发送者ID
  sender_type: String,      // seeker/taker
  type: String,             // text/image
  content: String,          // 文字内容或图片URL
  is_read: Boolean,         // 是否已读
  create_time: Date
}
```

### 5.3 优化措施

#### 5.3.1 数据库优化
- **索引优化**: 创建地理位置索引（2dsphere）
- **复合索引**: status + expire_time 复合索引
- **查询优化**: 限制单次返回数据量（分页）

#### 5.3.2 缓存策略
- **位置缓存**: 全局共享用户位置，减少重复获取
- **刷新间隔**: 首页和任务大厅30秒内不重复刷新
- **请求去重**: 请求ID机制，忽略过期请求结果

#### 5.3.3 实时通讯优化
- **优先watch**: 使用数据库watch实现实时推送
- **降级轮询**: watch失败时自动降级为3秒轮询
- **消息合并**: 批量处理未读消息

### 5.4 性能监控

| 监控项 | 告警阈值 | 监控方式 |
|--------|---------|---------|
| 云函数耗时 | > 500ms | 云开发控制台日志 |

## 六、安全规格

### 6.1 数据安全
- **用户隐私**: 不存储用户真实姓名、手机号
- **位置精度**: 使用gcj02坐标系，符合国内法规
- **数据加密**: 云数据库自动加密存储

### 6.2 内容安全
- **内容审核**: 接入微信msgSecCheck接口
- **聊天审核**: 文字消息实时安全检测
- **图片审核**: 聊天图片自动过滤

### 6.3 权限控制
- **身份验证**: 基于微信openid验证
- **操作权限**: 求助者/帮助者角色权限分离
- **数据隔离**: 用户只能访问自己的数据
