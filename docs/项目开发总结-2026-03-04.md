## 一、项目概述

### 1.1 产品定位
"问当地"是基于地理位置的即时互帮互助微信小程序，连接有异地实时信息需求的求助者与对应地点的帮助者。

**Slogan**: 当地想问啥，就来问当地

### 1.2 核心功能
- 用户双角色：可同时作为求助者（🙏）和帮助者（🤗）
- 基于地理位置的任务匹配（5公里范围）
- 即时聊天沟通
- 积分激励体系

### 1.3 技术栈
| 层级 | 技术 |
|------|------|
| 前端 | 微信小程序原生开发 |
| 后端 | 微信云开发（云函数 + 云数据库） |
| 存储 | 云存储（图片） |
| 地图 | 腾讯地图 SDK |

## 四、云函数清单

#### wdd-login
**功能**: 用户登录/注册
**入参**: 无（从 wx.getUserProfile 获取）

```javascript
{
  code: 0,
  data: {
    userInfo: { ... },
    points: { available, frozen, total },
    isNewUser: boolean,
    signInStatus: { canSignIn, consecutiveDays }
  }
```

**逻辑**:
1. 根据 _openid 查询用户
2. 不存在则创建新用户（送100积分）

## 六、核心业务逻辑

### 6.1 任务状态流转
```
┌──────────┐     接单      ┌──────────┐    确认完成    ┌──────────┐
│  pending │ ────────────→ │ ongoing  │ ────────────→ │completed │
│ (待匹配) │               │ (进行中) │               │ (已完成) │
└──────────┘               └──────────┘               └──────────┘
      │                                              ▲
      │ 超时/取消                                    │
      └──────────────────────────────────────────────┘
                    (已取消)
```

### 6.2 积分流转
```
用户A（求助者）                    用户B（帮助者）
    │                                   │
    │ 发布任务（冻结10积分）            │
    │ ────────────────────────────────→│
    │ 可用:90  冻结:10  总计:100        │
    │                                   │
    │                                   │ 接单
    │                                   │
    │ 确认完成                          │
    │ ────────────────────────────────→│
    │ 可用:90  冻结:0   总计:90         │ 可用:110 冻结:0 总计:110
```

### 6.3 地理位置处理
**位置数据格式**（GeoJSON）:
```javascript
{
  location: {
    name: "春熙路",
    type: "Point",
    coordinates: [104.0668, 30.6586] // [经度, 纬度]
  }
}
```

**距离计算**:
- 使用 Haversine 公式计算直线距离
- 单位：米（整数）
- 无效距离标识：99999999

- 云函数：`wdd-` + 功能名（如 `wdd-get-needs`）
- 组件：小驼峰（如 `loading`）

### 7.2 代码规范
- 使用 CSS 变量，禁止硬编码颜色，除非业务需要
- 页面必须使用公共组件（loading/empty/error）
- 云函数必须处理错误并返回标准格式
- API 调用前检查用户位置
- 严格按照微信小程序语法规范编写代码

### 7.3 标准响应格式
```javascript
// 成功
{ code: 0, data: {...}, message: "success" }

// 失败
{ code: -1, data: null, message: "错误信息" }
```
