# 问当地技术架构设计

**更新日期**：2026-05-14  
**适用版本**：当前小程序开发版本  
**技术形态**：微信小程序原生开发 + 微信云开发

## 1. 架构总览

```text
用户
  ↓
微信小程序前端
  - 页面：发布、任务大厅、详情、聊天、我的、钱包、举报、申诉
  - 组件：空状态、错误状态、加载状态
  - 工具：日期、日志、隐私授权、金额展示、平台规则、聊天缓存
  ↓
微信云开发
  - 云函数：登录、支付、任务、聊天、通知、结算、提现、工单
  - 云数据库：wdd-* 业务集合
  - 云存储：任务图片、聊天图片、用户相关图片
  ↓
外部能力
  - 微信登录
  - 微信支付
  - 微信隐私授权
  - 微信位置能力
  - 腾讯地图相关能力
```

## 2. 前端结构

前端位于 `miniprogram` 目录，采用微信小程序原生技术栈：

| 类型 | 位置 | 说明 |
| --- | --- | --- |
| 全局入口 | `miniprogram/app.js` | 初始化云开发、登录状态、位置、消息监听、平台配置 |
| 全局样式 | `miniprogram/app.wxss` | 定义主色、字体、按钮、卡片、标签、动画和工具类 |
| 页面 | `miniprogram/pages/*` | 各业务页面 |
| 组件 | `miniprogram/components/*` | 通用加载、空状态、错误状态 |
| 工具 | `miniprogram/utils/*` | 日志、日期、隐私、缓存、金额、平台规则 |
| 配置 | `miniprogram/config/types.js` | 举报类型、状态映射、类型映射 |

## 4. 云函数分工

| 云函数 | 主要职责 |
| --- | --- |
| `wdd-login` | 登录注册、更新用户信息、帮助者资料 |
| `wdd-payment` | 创建支付订单、确认支付、创建任务、查询订单、退款 |
| `wdd-get-needs` | 获取任务大厅、我的求助、我的任务、任务详情、评价详情 |
| `wdd-take-need` | 帮助者接单，创建接单记录，更新任务状态 |
| `wdd-chat` | 获取任务聊天信息、拉取消息、发送消息、已读处理 |
| `wdd-notify` | 会话列表、系统通知、未读数、角标数据 |
| `wdd-settlement` | 完成任务、取消任务、提交评价、写入结算记录 |
| `wdd-withdraw` | 发起提现、查询提现记录、执行商家转账、处理回调 |
| `wdd-get-point-records` | 查询积分记录和余额记录 |
| `wdd-sign-in` | 每日签到、积分奖励、签到记录 |
| `wdd-report` | 提交举报、撤销举报、补充材料、查询举报详情 |
| `wdd-appeal` | 提交申诉、撤销申诉、补充材料、查询申诉详情 |
| `wdd-ticket` | 客服工单列表、详情、裁定、超时处理 |
| `wdd-auto-cancel` | 自动取消过期任务、自动完成超时任务、处理异常退款/转账 |
| `wdd-get-config` | 获取平台配置，如费率、积分规则、提现规则 |
| `wdd-db-init` | 数据库字段初始化和旧数据兼容迁移 |
| `wdd-geo` | 地理位置相关能力 |
| `wdd-msg-callback` | 订阅/回调类消息处理 |

## 5. 核心业务流程

### 5.1 登录流程

```text
用户授权微信资料
  ↓
前端调用 wdd-login
  ↓
云函数按 openid 查询用户
  ↓
不存在则创建用户、赠送新用户积分
  ↓
返回 userInfo，前端写入 globalData 和本地缓存
```

### 5.2 发布求助与支付流程

```text
填写求助信息
  ↓
调用 wdd-payment.createOrder 创建支付订单
  ↓
拉起微信支付
  ↓
支付成功后调用 wdd-payment.confirmPayment
  ↓
云函数创建 wdd-needs 任务
  ↓
任务进入 pending 待匹配
```

### 5.3 接单流程

```text
帮助者查看任务详情
  ↓
调用 wdd-take-need
  ↓
事务中检查任务仍为 pending
  ↓
任务改为 ongoing，写入 wdd-need-takers
  ↓
发送系统通知，进入聊天
```

### 5.4 聊天流程

```text
进入聊天页
  ↓
wdd-chat.getTaskInfo 校验双方权限
  ↓
wdd-chat.getMessages 拉取历史消息
  ↓
wdd-chat.sendMessage 发送文字/图片
  ↓
写入 wdd-messages，并更新未读通知
```

### 5.5 完成与结算流程

```text
求助者确认完成或系统自动完成
  ↓
wdd-settlement.completeTask
  ↓
计算平台服务费和帮助者收入
  ↓
任务改为 completed
  ↓
帮助者余额增加
  ↓
写入 wdd-balance-records 和通知
```

### 5.6 举报/申诉工单流程

```text
用户提交举报或申诉
  ↓
写入 wdd-reports 或 wdd-appeals
  ↓
创建 wdd-tickets 工单
  ↓
任务状态改为 breaking
  ↓
客服裁定
  ↓
任务改为 completed 或 cancelled
  ↓
按裁定结果处理资金和通知
```

## 6. 全局配置

平台规则通过 `wdd-config` 集合的 `platform` 文档配置，再由 `wdd-get-config` 拉取到前端和云函数中使用。

建议配置内容包括：

1. 平台服务费率。
2. 最低发布金额。
3. 最低提现金额。
4. 签到积分规则。
5. 邀请奖励规则。
6. 工单补充材料时限。
7. 客服人员 OpenID 白名单：`customer_service_openids`。

## 7. 安全设计

| 风险 | 当前设计 |
| --- | --- |
| 用户伪造身份 | 云函数通过微信上下文 `OPENID` 获取当前用户 |
| 越权查看任务/聊天 | 云函数校验任务发布者、接单者身份 |
| 重复接单 | 接单时在事务中检查任务状态 |
| 重复评价 | 根据任务、评价人、评价对象检查评价记录 |
| 资金错算 | 支付、结算、提现使用订单和余额记录留痕 |
| 争议处理 | 举报/申诉后任务进入 `breaking`，由客服裁定 |
| 客服权限滥用 | 通过 `wdd-config/platform.customer_service_openids` 判断客服身份 |
| 隐私授权 | `app.json` 声明位置权限，`app.js` 处理隐私授权 |

## 8. 性能设计

1. 小程序启用 `lazyCodeLoading`，减少首屏加载压力。
2. 全局缓存用户信息、当前位置和平台配置。
3. 聊天页使用本地缓存和数据库监听，减少重复拉取。
4. 任务列表采用分页查询。
5. 任务大厅可按状态、类型、距离、时间和金额筛选，建议为高频查询字段建立索引。

## 9. 部署结构

| 部分 | 部署方式 |
| --- | --- |
| 小程序前端 | 微信开发者工具上传体验版/正式版 |
| 云函数 | 微信开发者工具或云开发控制台逐个上传部署 |
| 云数据库 | 云开发控制台创建集合、索引和权限 |
| 云存储 | 云开发控制台配置存储权限和安全规则 |
| 微信支付 | 微信支付商户号、回调、证书和商家转账配置 |

## 10. 当前技术债与建议

1. 建议建立统一的后台管理端，减少客服直接依赖小程序页面或数据库。
2. 建议把状态枚举集中维护，避免前端、云函数、文档三处不一致。
3. 建议补齐云函数自动化测试，尤其是支付、退款、提现、工单裁定。
4. 建议为生产环境增加操作审计和异常告警。
