# 问当地小程序 - 数据刷新逻辑梳理

**文档日期**: 2026-03-04

---

## 一、全局刷新策略

### 1.1 位置获取策略

| 时机 | 操作 | 说明 |
|------|------|------|
| 小程序启动 | `app.updateUserLocation()` | 全局获取，不阻塞启动流程 |
| 页面显示 | `onShow()` 中刷新 | 用户可能移动了位置 |
| 下拉刷新 | 重新获取位置 | 确保使用最新位置计算距离 |

### 1.2 登录状态检查

| 页面 | 检查时机 | 未登录处理 |
|------|---------|-----------|
| 所有页面 | `onLoad()` | 提示登录并返回 |
| 需要登录的页面 | `onShow()` | 检查状态变化 |

## 二、各页面刷新逻辑详情

### 2.1 首页（index）

#### 刷新时机

| 触发方式 | 实现方法 | 前置条件 |
|---------|---------|---------|
| 页面加载 | `onLoad()` → `loadNearbyNeedsWithLocation()` | 已登录 |
| 页面显示 | `onShow()` → `refreshLocationAndLoad()` | 已登录 |
| 下拉刷新 | `onPullDownRefresh`（需添加） | 已登录 |

#### 实现方式

```javascript
onLoad() {
  this.checkLoginStatus()
  this.loadNearbyNeedsWithLocation()  // 确保位置后加载
}

onShow() {
  if (this.data.isLoggedIn) {
    this.refreshLocationAndLoad()  // 刷新位置并加载
  }
}
```

#### 前置条件
- 用户已登录
- 已获取用户位置（若无则尝试获取）

#### 刷新效果
- 显示5公里内的待匹配任务
- 按时间排序（最新优先）
- 显示距离信息

---

### 2.2 任务大厅（task-hall）

#### 刷新时机

| 触发方式 | 实现方法 | 前置条件 |
|---------|---------|---------|
| 页面加载 | `onLoad()` → `syncUserLocationFromGlobal()` + `loadTasks()` | 无 |
| 页面显示 | `onShow()` → `syncUserLocationFromGlobal()` + `loadTasks(true)` | 无 |
| 下拉刷新 | `onRefresh()` | 无 |
| 上拉加载 | `onLoadMore()` | 有下一页 |
| 筛选切换 | `selectFilter()` / `selectDistance()` / `selectSort()` | 无 |

#### 实现方式

```javascript
onLoad() {
  this.syncUserLocationFromGlobal()  // 从全局同步位置
  this.loadTasks()  // 加载任务
}

onShow() {
  this.syncUserLocationFromGlobal()  // 每次显示刷新位置
  this.loadTasks(true)  // 强制刷新
}

// 下拉刷新
onRefresh() {
  this.syncUserLocationFromGlobal()
  this.loadTasks(true)  // 重置到第一页
}

// 上拉加载更多
onLoadMore() {
  if (!this.data.hasMore) return
  this.setData({ page: this.data.page + 1 })
  this.loadTasks(false)  // 追加数据
}
```

#### 前置条件
- 位置：从全局获取，失败也可加载（距离显示为--）
- 登录：不需要登录即可浏览

#### 刷新效果
- 支持距离筛选（1/3/5/10公里/全部）
- 支持排序（距离/积分/时间）
- 支持类型筛选
- 分页加载（每页10条）

### 2.3 我的求助（my-needs）

#### 刷新时机

| 触发方式 | 实现方法 | 前置条件 |
|---------|---------|---------|
| 页面加载 | `onLoad()` → `loadNeeds()` | 已登录 |
| 页面显示 | `onShow()` → 检查刷新标记 | 已登录 |
| 下拉刷新 | `onPullDownRefresh()` | 已登录 |
| 上拉加载 | `loadMore()` | 有下一页 |
| 筛选切换 | `switchFilter()` | 已登录 |

#### 实现方式

```javascript
onLoad() {
  if (!app.globalData.isLoggedIn) {
    // 未登录，提示并返回
    return
  }
  this.loadNeeds()
}

onShow() {
  // 检查是否需要刷新（从详情页返回等）
  if (wx.getStorageSync('refreshMyNeeds')) {
    wx.removeStorageSync('refreshMyNeeds')
    this.setData({ page: 1, needs: [] })
    this.loadNeeds()
  }
}

onPullDownRefresh() {
  this.setData({ page: 1, needs: [] })
  this.loadNeeds().then(() => {
    wx.stopPullDownRefresh()
  })
}
```

#### 前置条件
- 必须已登录
- 只查询自己的任务

#### 刷新效果
- 按状态筛选（全部/进行中/已完成/已取消）
- 分页加载
- 显示任务状态、剩余时间、地点

### 2.4 已接任务（my-tasks）

#### 刷新时机
与"我的求助"类似：
- `onLoad()` 加载
- `onShow()` 检查刷新标记
- `onPullDownRefresh()` 下拉刷新
- `loadMore()` 上拉加载

#### 特殊点
- 查询的是 `need_takers` 表关联的任务
- 显示作为帮助者的任务列表

### 2.5 任务详情（task-detail）

#### 刷新时机

| 触发方式 | 实现方法 | 前置条件 |
|---------|---------|---------|
| 页面加载 | `onLoad()` → `loadTaskDetail()` | 有任务ID |
| 页面显示 | `onShow()` → `loadTaskDetail()` | 有任务ID |

#### 实现方式

```javascript
onLoad(options) {
  const { id } = options
  if (!id) { /* 错误处理 */ return }
  this.setData({ needId: id })
  this.loadTaskDetail()
}

onShow() {
  if (this.data.needId) {
    this.loadTaskDetail()  // 每次显示都刷新
  }
}
```

#### 前置条件
- 必须有任务ID
- 不需要登录（可浏览）

#### 刷新效果
- 获取最新任务状态
- 判断用户身份（求助者/帮助者/旁观者）
- 显示可操作按钮（接单/完成/取消等）

### 2.6 聊天页（chat）

#### 刷新时机

| 触发方式 | 实现方法 | 前置条件 |
|---------|---------|---------|
| 页面加载 | `onLoad()` → 初始化 | 有任务ID |
| 页面显示 | `onShow()` → 开始轮询 | 已登录 |
| 页面隐藏 | `onHide()` → 停止轮询 | - |
| 实时推送 | `watch()` 监听 | 建立连接后 |
| 定时轮询 | `startPolling()` | watch失败时 |

#### 实现方式

```javascript
onLoad(options) {
  const { needId, isSeeker } = options
  this.setData({ needId, isSeeker })
  this.loadTaskInfo()      // 加载任务信息
  this.loadHistoryMessages() // 加载历史消息
}

onShow() {
  this.startPolling()  // 开始轮询新消息
  this.markMessagesRead() // 标记已读
}

onHide() {
  this.stopPolling()   // 停止轮询
}
```

#### 前置条件
- 必须是求助者或帮助者
- 任务状态为"进行中"

#### 刷新效果
- 实时显示新消息
- 显示任务信息（求助位置、描述等）
- 显示对方资料

### 2.7 消息中心（messages）

| 页面显示 | `onShow()` → `loadSessions()` | 已登录 |
| 下拉刷新 | 需添加 | 已登录 |

#### 实现方式

```javascript
onLoad() {

onShow() {

```

#### 前置条件

#### 刷新效果

- 显示最后一条消息
- 显示未读消息数

## 三、数据刷新标记机制

### 3.1 使用 Storage 标记刷新

| 标记键 | 设置位置 | 消费位置 | 用途 |
|--------|---------|---------|------|
| `refreshMyNeeds` | 发布/取消/完成任务后 | my-needs `onShow()` | 刷新我的求助列表 |
| `refreshMyTasks` | 接单/完成任务后 | my-tasks `onShow()` | 刷新已接任务列表 |

### 3.2 使用全局数据

| 数据 | 存储位置 | 使用位置 | 用途 |
|------|---------|---------|------|
| `userLocation` | app.globalData | 各页面 | 共享用户位置 |
| `userInfo` | app.globalData | 各页面 | 共享用户信息 |
| `isLoggedIn` | app.globalData | 各页面 | 登录状态 |

## 四、刷新优化建议

### 4.1 当前存在的问题

1. ~~**重复刷新**：首页和任务大厅每次 `onShow()` 都刷新，可能过于频繁~~ ✅ 已修复
2. **缺少缓存**：任务详情每次进入都重新加载，没有本地缓存
3. **无节流控制**：快速切换页面可能导致多次请求

### 4.2 已完成的优化

#### 首页（index）刷新优化
- **刷新间隔控制**：30秒内不重复刷新
- **强制刷新机制**：发布任务后设置 `forceRefreshIndex` 标记
- **时间戳记录**：`lastRefreshTime` 记录上次刷新时间

#### 任务大厅（task-hall）刷新优化
- **刷新间隔控制**：30秒内不重复刷新
- **强制刷新机制**：发布任务后设置 `forceRefreshTaskHall` 标记
- **筛选切换立即刷新**：切换筛选/排序/距离时不受时间限制

### 4.3 待完成的优化

1. **增加刷新间隔**：
   - 页面显示时，如果距离上次刷新 < 30秒，不刷新
   - 只在必要时强制刷新（如下拉）

2. **本地缓存**：
   - 任务详情可缓存 5分钟
   - 用户资料可缓存 1分钟

3. **请求合并**：
   - 多个并行请求合并为一个云函数调用

4. **增量更新**：
   - 消息使用 `watch()` 监听，减少轮询频率
