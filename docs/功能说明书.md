# 问当地小程序功能说明书

**更新日期**：2026-05-14  
**适用版本**：当前小程序开发版本  
**文档目标**：用产品语言说明“问当地”当前要交付给用户的功能范围和业务规则。

## 1. 产品定位

“问当地”解决的是用户无法亲临现场、但又需要快速获得某地实时信息的问题。例如：目的地是否下雨、前方路段是否拥堵、店铺是否营业、停车场是否还有空位、现场排队是否严重。

产品通过“地理位置匹配 + 悬赏激励 + 即时聊天”的方式，让求助者发布问题，让当地或附近的帮助者接单回答。

## 2. 用户角色

| 角色 | 说明 | 主要能力 |
| --- | --- | --- |
| 求助者 | 有异地实时信息需求的人 | 发布求助、支付悬赏、聊天沟通、确认完成、评价、举报/申诉 |
| 帮助者 | 能提供当地实时信息的人 | 浏览任务、接单、聊天回答、获得收益、评价、举报/申诉 |
| 客服/管理员 | 处理争议和异常工单的人 | 查看举报/申诉工单、补充材料、裁定任务、处理资金分配 |

同一个普通用户可以同时是求助者和帮助者。

## 3. 页面与导航

底部导航包含 4 个入口：

| 导航 | 页面 | 说明 |
| --- | --- | --- |
| 首页 | `pages/index/index` | 展示产品入口、快捷发布、附近任务等核心信息 |
| 大厅 | `pages/task-hall/task-hall` | 浏览可接任务，按距离、类型、排序筛选 |
| 消息 | `pages/messages/messages` | 查看聊天会话和系统通知 |
| 我的 | `pages/my/my` | 查看个人资料、我的求助、我的任务、钱包、记录、设置 |

非底部导航页面包括发布求助、任务详情、聊天、钱包、提现、评价、举报、申诉、客服工单、公开资料等。

## 4. 用户与资料功能

### 4.1 微信登录

用户通过微信授权登录。系统根据微信 `openid` 识别用户身份，首次登录创建用户档案，并赠送新用户积分。

### 4.2 个人资料

用户可维护头像、昵称等基础信息。帮助者可配置帮助者资料，包括常去地点、帮助意愿和可帮助类型，用于提高任务匹配质量。

### 4.3 信用与评价

用户拥有信用分、平均评分、评价数量等信誉信息。任务完成后双方可以互评，评价会影响用户公开资料的可信度。

## 5. 求助发布功能

### 5.1 求助类型

当前支持的核心求助场景包括：

1. 实时天气
2. 道路拥堵
3. 店铺营业
4. 停车空位
5. 排队情况
6. 其他本地实时信息

### 5.2 发布内容

发布求助时，用户需要填写：

| 内容 | 说明 |
| --- | --- |
| 求助类型 | 选择最接近的问题类型 |
| 求助地点 | 可使用地图选点或手动补充地点名称 |
| 问题描述 | 描述想知道什么，建议具体、可回答 |
| 悬赏金额 | 通过微信支付完成支付后，任务正式发布 |
| 有效期 | 支持 30 分钟、1 小时、2 小时、4 小时、12 小时、24 小时 |
| 图片 | 可上传现场、店铺、路线等辅助图片 |

### 5.3 支付与发布

发布流程采用“先创建支付订单，再确认支付并创建任务”的方式。支付成功后，任务进入 `pending` 待匹配状态，并出现在任务大厅。

## 6. 任务大厅与接单

### 6.1 任务浏览

帮助者可在任务大厅浏览待匹配任务，任务信息包含类型、地点、距离、悬赏、发布时间、有效期和描述摘要。

### 6.2 筛选与排序

当前支持按距离、类型、时间、悬赏等维度进行筛选或排序。距离计算依赖用户当前位置。

### 6.3 接单规则

帮助者点击接单后，系统会检查任务是否仍为待匹配状态。接单成功后：

1. 任务状态从 `pending` 变为 `ongoing`。
2. 创建接单记录。
3. 求助者和帮助者可进入聊天。
4. 系统向相关用户发送通知。

## 7. 聊天与消息

### 7.1 聊天范围

聊天围绕具体任务展开，只有任务相关的求助者和帮助者可以进入对应会话。

### 7.2 消息类型

当前支持文字和图片消息。消息会记录发送者、接收者、任务、内容、已读状态和发送时间。

### 7.3 实时提醒

小程序启动后会监听未读消息。收到新消息时，消息 Tab 会显示角标。若实时监听失败，聊天页可通过接口刷新消息列表。

## 8. 任务状态与结算

### 8.1 任务状态

| 状态 | 中文名 | 说明 |
| --- | --- | --- |
| `pending` | 待匹配 | 已发布，等待帮助者接单 |
| `ongoing` | 进行中 | 已有人接单，双方沟通中 |
| `completed` | 已完成 | 任务正常完成或被裁定完成 |
| `cancelled` | 已取消 | 任务取消、超时取消或被裁定取消 |
| `breaking` | 审核中 | 出现举报/申诉，等待客服裁定 |

### 8.2 正常完成

任务完成后，系统按平台配置计算服务费和帮助者收入，将帮助者收入记入钱包余额，并生成余额记录。

### 8.3 取消与超时

待匹配任务在有效期内无人接单或由发布者取消时，可进入取消流程。已支付订单需要配合退款流程处理。

进行中的任务超过规则时间后，可由系统自动完成或提醒用户处理，具体执行由自动任务云函数负责。

## 9. 钱包、余额与提现

### 9.1 钱包

帮助者完成任务后获得的收入进入余额。钱包页展示可用余额、冻结金额、累计收入、累计提现等信息。

### 9.2 余额记录

余额变动会写入 `wdd-balance-records`，包括任务收入、提现、裁定退款等。

### 9.3 提现

用户可从钱包发起提现。提现记录会经历处理中、转账中、完成或失败等状态。实际转账依赖微信支付商家转账能力和后台配置。

## 10. 积分、签到与邀请

积分用于平台内权益体系，不等同现金，不可提现。当前积分来源包括新用户赠送、签到奖励、邀请奖励等。积分记录写入 `wdd-point-records`，签到记录写入 `wdd-sign-in-records`。

## 11. 举报、申诉与客服工单

### 11.1 举报

当用户认为对方存在虚假信息、失联、诱导私下交易、辱骂骚扰、恶意拖延等行为时，可以发起举报。

举报创建后：

1. 写入 `wdd-reports`。
2. 创建客服工单 `wdd-tickets`。
3. 任务进入 `breaking` 审核中。
4. 用户可撤销举报或补充材料。

### 11.2 申诉

当用户对任务处理、金额结算、举报结果或对方行为有异议时，可以发起申诉。

申诉创建后：

1. 写入 `wdd-appeals`。
2. 创建客服工单 `wdd-tickets`。
3. 任务进入 `breaking` 审核中。
4. 用户可撤销申诉或补充材料。

### 11.3 客服裁定

客服可对工单进行裁定，结果包括：

| 裁定 | 任务结果 | 资金结果 |
| --- | --- | --- |
| 取消任务 | `cancelled` | 悬赏退回求助者 |
| 完成任务 | `completed` | 悬赏结算给帮助者 |
| 部分完成 | `completed` | 按比例分配给求助者和帮助者 |

## 12. 当前边界与后续增强

当前版本的重点是完成真实交易闭环。后续可以增强：

1. 更精确的地理围栏和帮助者真实性验证。
2. 管理后台可视化，减少直接操作云函数和数据库。
3. 更完整的内容安全、图片审核和风控策略。
4. 任务推荐算法和帮助者等级体系。
5. 积分权益兑换功能。
