# 问当地小程序 - 开发规范

**创建日期**: 2026-03-04
**适用范围**: 所有与任务地理位置相关的功能开发

## 一、地理位置数据规范（核心）

### 1.1 所有任务必须包含 location 字段

**强制要求**：所有在 `wdd-needs` 集合中的任务文档，必须包含完整的 `location` 字段。

**规范格式**：
```javascript
{
  location: {
    name: "春熙路",           // 地点名称（展示用）
    latitude: 30.6586,       // 纬度（-90 到 90）
    longitude: 104.0668      // 经度（-180 到 180）
  }
}
```

**验证规则**：
- `latitude` 和 `longitude` 必须为数字类型
- `name` 不能为空字符串
- 发布任务时前端必须校验位置已选择

**错误处理**：
```javascript
// 云函数中必须处理无效位置
if (!item.location || !item.location.latitude || !item.location.longitude) {
  // 标记为无效距离，不参与距离计算
  item.distance = 99999999  // 表示无效
}
```

## 二、数据库索引规范

### 2.1 索引创建标准

**新增索引必须提供以下信息**：
- **索引名称**：统一命名格式 `idx_字段名_类型`
- **是否唯一**：是 / 否
- **索引字段**：字段名及排序方式
- **索引类型**：普通索引 / 复合索引 / 地理位置索引（2dsphere）
- **用途说明**：该索引用于优化什么查询场景

### 2.2 现有索引清单

#### 索引1：地理位置索引（已创建）

| 属性 | 值 |
|------|-----|
| **索引名称** | `idx_location_2dsphere` |
| **是否唯一** | 否 |
| **索引字段** | `location` |
| **排序方式** | `2dsphere`（地理位置专用） |
| **用途** | 支持基于地理位置的查询和距离计算 |

#### 索引2：过期时间索引（已存在）

| 属性 | 值 |
|------|-----|
| **索引名称** | `idx_expire` |
| **是否唯一** | 否 |
| **索引字段** | `expire_time`（降序） |
| **排序方式** | `-1`（降序） |
| **用途** | 加速过期任务查询和清理 |

#### 索引3：状态+过期时间复合索引（待创建）

| 属性 | 值 |
|------|-----|
| **索引名称** | `idx_status_expire` |
| **是否唯一** | 否 |
| **索引字段** | `status`（升序）、`expire_time`（降序） |
| **排序方式** | `1`（升序）、`-1`（降序） |
| **用途** | 加速"查询待匹配且未过期任务"的常用查询 |

### 2.3 后续新增索引规范

**命名规则**：
- 单字段索引：`idx_字段名`（如 `idx_user_id`）
- 复合索引：`idx_字段1_字段2`（如 `idx_status_create_time`）
- 地理位置索引：`idx_字段名_2dsphere`（如 `idx_location_2dsphere`）

**创建时机**：
- 数据量 < 1万条：随时可创建
- 数据量 1-10万条：选择低峰期创建
- 数据量 > 10万条：必须评估影响，可能需要分步创建

**必需索引场景**：
- 查询条件中的字段
- 排序字段
- 地理位置查询字段
- 外键关联字段

## 三、用户位置获取规范

### 2.1 所有任务功能必须先获取位置

**核心原则**："问当地"是基于地理位置的互帮互助平台，**没有地理位置，任务无法准确匹配**。

**必须获取位置的场景**：
- [x] 首页附近求助（限制5公里内）
- [x] 任务大厅列表（距离排序/筛选）
- [ ] 发布求助（记录求助地点）
- [ ] 接单匹配（计算帮助者与任务距离）
- [ ] 消息/聊天（可选，用于实时距离更新）

### 2.2 位置获取流程

**标准封装方法**：
```javascript
// 确保有位置，如果没有则尝试获取
async ensureLocation() {
  let location = app.getUserLocation()
  if (!location) {
    try {
      await app.updateUserLocation()
      location = app.getUserLocation()
    } catch (err) {
      // 引导用户开启权限
      this.showLocationAuthGuide()
      return null
    }
  }
  return location
}
```

**权限引导**：
```javascript
showLocationAuthGuide() {
  wx.showModal({
    title: '需要位置权限',
    content: '问当地需要您的位置信息来匹配附近的任务和帮助者',
    confirmText: '去设置',
    success: (res) => {
      if (res.confirm) {
        wx.openSetting()
      }
    }
  })
}
```

### 2.3 位置更新时机

| 时机 | 操作 | 说明 |
|------|------|------|
| 小程序启动 | `app.updateUserLocation()` | 全局获取，不阻塞 |
| 页面显示 | `onShow()` 中刷新 | 用户可能移动了 |
| 下拉刷新 | 重新获取位置 | 确保最新位置 |
| 发布任务 | 必须获取当前位置 | 用于地图中心定位 |

## 四、云函数接口规范

### 3.1 获取任务列表接口

**云函数**: `wdd-get-needs`

**必传参数**：
```javascript
{
  latitude: 30.6586,    // 用户当前纬度
  longitude: 104.0668,  // 用户当前经度
  distance: 5000,       // 筛选距离（米），0表示不限
  filter: 'all',        // 类型筛选
  sort: 'distance'      // 排序方式
}
```

**注意**：即使不传 `latitude/longitude`，只要有 `distance` 参数，也要执行距离筛选（基于任务已有距离数据）。

### 3.2 距离计算规范

**计算方式**：使用 Haversine 公式计算直线距离

**单位**：米（统一使用整数）

**精度**：四舍五入到整数米

**无效距离标识**：
- `99999999` 表示无效/无穷大距离
- 前端展示时转换为 `--`

## 五、性能优化预留

### 4.1 当前架构限制

**已知瓶颈**：
- 全表查询后内存计算距离
- 当任务量 > 1000 时可能影响性能

**监控指标**：
- 云函数执行时间 > 500ms 时告警
- 单次查询返回数据量 > 100 条时告警

### 4.2 未来优化方向

**阶段1：数据库索引（部分完成）**

已存在索引：
- `idx_expire`：过期时间索引

待创建索引：
- `idx_location_2dsphere`：地理位置索引（优先级高）
- `idx_status_expire`：状态+过期时间复合索引（优先级中）

如需新增索引，参考"二、数据库索引规范"。

**阶段2：地理网格分片**
- 将成都划分为 1km × 1km 网格
- 任务增加 `grid_id` 字段
- 查询时只查当前网格+周围8格

**阶段3：Redis 缓存**
- 缓存热门区域任务ID列表
- 按区域预计算距离排序
