# 意见反馈功能说明

## 功能范围

### 用户侧

- 入口：`我的 → 其他 → 意见反馈`
- 支持填写标题、反馈内容和最多 3 张选填图片。
- 标题限制为 2～40 字，内容限制为 5～500 字。
- 页面展示当日剩余提交次数。
- 提交次数按北京时间自然日计算，限制值读取 `wdd-config/platform.feedback_daily_limit`，默认 3 条。
- 表单会自动保存本地草稿，提交失败时不会清空。
- 用户可从提交页右上角进入“我的反馈”，查看历史列表和只读详情。
- 提交成功后会生成一条“意见反馈已收到”的站内系统通知。
- 客服处理后暂不向用户发送通知。

### 客服侧

- 入口：`我的 → 客服系统 → 意见反馈处理`
- 列表分为“待处理”和“已处理”两个 Tab。
- 卡片展示标题、内容摘要、提交用户、提交时间和图片数量。
- 详情页只读展示反馈内容、图片、提交用户、设备环境和处理信息。
- 待处理详情页底部提供“已知晓”按钮，确认后进入已处理状态。
- 多名客服并发处理同一条反馈时，仅第一位客服可以成功，其他客服会看到“该反馈已由其他客服处理”。

## 数据与隐私

- 反馈记录集合：`wdd-feedbacks`
- 每日额度集合：`wdd-feedback-daily-quotas`
- 用户提交次数和反馈创建处于同一事务中，避免并发突破限制。
- 反馈记录不提供用户删除接口，作为平台运营记录保留。
- 客服接口不会返回用户 OpenID，只返回昵称和头像。
- 用户注销后，用户资料会按现有注销逻辑显示为“已注销用户”，反馈记录不会被物理删除。

## 配置项

在超级管理员“系统配置 → 客服配置”中可调整：

```text
feedback_daily_limit
```

允许范围为 1～20，默认值为 3。

## 部署要求

1. 部署云函数 `wdd-feedback`，并安装云端依赖。
2. 重新部署以下已修改云函数：
   - `wdd-get-config`
   - `wdd-config-admin`
   - `wdd-db-init`
   - `wdd-temp-clear-all`（仅运维清理工具需要时部署）
3. 在云数据库创建集合：
   - `wdd-feedbacks`
   - `wdd-feedback-daily-quotas`
4. 为 `wdd-feedbacks` 建议建立以下联合索引：
   - `user_id` 升序、`create_time` 降序
   - `status` 升序、`create_time` 降序
   - `status` 升序、`resolve_time` 降序
5. 上传并编译小程序代码。

## 当前明确暂缓

- 标题、内容和图片的内容安全检测。
- 客服处理意见反馈后的用户通知。
- 客服文字回复、内部备注、标签、优先级和指派。

内容安全检测虽然本版暂缓，但在公开发布前仍建议结合微信平台审核要求再次评估。
