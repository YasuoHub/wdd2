# 问当地 UI 设计图生成提示词合集

**用途**：把本文中的「全局提示词」和某个「页面提示词」一起复制给 GPT-image2 或其他图像生成 AI，用于生成统一风格的微信小程序页面设计图。

**使用方式**：

1. 每次只生成一个页面，稳定性和细节更好。
2. 先复制「全局提示词」，再追加某个页面的提示词。
3. 所有页面都按微信小程序移动端竖屏设计，宽度按 750rpx 理解。
4. 如果生成图中文字不准，优先要求模型重画该页面，不要一次性让它改多个页面。
5. 生成页面整体效果图后，明显需要切图复用的图标和背景图必须作为独立图片单独生成；如果图像工具不能一次输出多张独立图片，就按资产逐张生成。

## 0. 全局提示词

```text
设计一个微信小程序移动端 UI 页面，产品名为“问当地”。

产品定位：
“问当地”是一个基于地理位置的实时信息互助小程序。用户可以发布带悬赏的本地求助，例如询问某地是否下雨、路况是否拥堵、店铺是否营业、停车场是否有空位、现场排队是否严重；附近的帮助者可以在任务大厅接单，通过聊天提供实时信息并获得收益。

整体 UI 风格：
清爽、可信、即时、有地点感、轻量交易、本地互助。界面要明亮干净，亲和但不幼稚，有真实可发布产品的完成度。不要做成营销落地页，不要重插画，不要炫酷科技风，不要像外卖抢单平台那样刺激，也不要像金融 App 那样沉重。

设计规格：
移动端微信小程序界面，画布比例为手机竖屏，按 750rpx 宽度设计。保留微信小程序顶部导航区域和底部 Tab 样式。页面背景使用极浅蓝灰色 #F5FAFD，卡片使用白色 #FFFFFF。

颜色规范：
主色使用天蓝色 #5DB8E6，用于主按钮、选中状态、重点图标。
深蓝 #2A7DB3 用于标题强调和链接。
浅蓝 #A8D8F0 用于弱背景和轻提示。
橙色 #FF8C69 用于悬赏金额、支付、重要提醒。
绿色 #6DD5B0 用于完成、收益、成功状态。
黄色 #FFD166 用于等待、积分、温和提示。
红色 #FF6B6B 用于举报、错误、风险。
主文字 #2C3E50，次文字 #4A6572，弱文字 #6B8BA4，边框 #E6EEF3。

排版规范：
页面左右边距 32rpx。卡片圆角 16rpx，按钮使用胶囊圆角。使用轻微浅蓝灰阴影，不要厚重投影。字体使用系统中文字体，页面标题约 40rpx，模块标题约 32rpx，正文约 28rpx，辅助文字约 24rpx。当前代码中部分入口使用 emoji 表达图标语义，设计图可以按相同语义替换为统一线性图标。

组件规范：
任务卡片必须突出地点、问题、悬赏、状态、剩余时间。状态标签颜色固定：待匹配浅蓝或黄色，进行中天蓝，已完成绿色，已取消灰色，审核中橙色。金额类信息用橙色或深色加粗。涉及钱包、提现、举报、申诉、客服裁定的页面要更稳重，减少装饰。

文字与图标规范：
页面上的文字要克制，只保留用户完成操作必须理解的信息。不要写大段说明，不要堆功能介绍，不要在页面里解释产品理念。所有文字都要短、直接、可执行。

在意思表达清楚的前提下，优先使用图标、状态标签、数字、颜色和布局来传达信息。功能入口、任务类型、筛选项、状态、地点、时间、金额、消息、钱包、举报、申诉、评价等都应使用统一线性图标辅助表达。图标语义要与当前小程序入口一致，风格要一致，线宽统一，不能混用卡通图标或复杂插画。

每个页面只突出 1 个主要目标。首屏不要出现太多解释文字。列表页优先展示“图标 + 标签 + 数字 + 简短摘要”。表单页优先使用图标选择项、chips、分段控件、开关、上传占位框，而不是长文本说明。

文案长度要控制：按钮文案 2-6 个字，例如“发布”“接单”“提现”“提交”“查看详情”“进入聊天”；状态文案 2-4 个字，例如“待接单”“进行中”“已完成”“审核中”；提示文案最多一行，能不用就不用。必须说明规则时，优先使用“规则”入口或可展开说明，不要直接铺满页面。

避免大段说明文字、长标题、密集操作说明、每个模块都配解释句、以及图标完全无文字辅助导致用户无法理解。

效果图与切图资产输出规范：
除了生成当前页面的整体 UI 效果图，还需要把页面中明显需要切图复用的图标和背景图直接生成出来，作为可直接用于小程序开发的独立 PNG 资产。

拆分规则：列表里的每一项图标必须独立生成一张图片，严禁把多个图标合并在同一张画布或同一张 icon sheet 中。AI 输出即为切图完成状态，不需要后期分割、裁剪或抠图。

图标规格：每个图标输出为 80×80px 的 2 倍高清 PNG 图片，透明 Alpha 背景，无任何白色底色，无阴影外溢，无多余边距。图标主体居中，适配移动端小程序小尺寸显示。

背景图规格：需要独立切图的装饰背景、顶部背景、空状态背景、地图占位背景等，按设计中实际使用尺寸的 2 倍高清 PNG 输出，透明 Alpha 背景，无任何白色底色，无阴影外溢。背景图必须可直接叠加到小程序页面中，不要带整页 UI 截图。

视觉要求：图标使用细线条 UI 风格，线宽统一，圆角规整，统一配色，优先使用主色 #5DB8E6、深蓝 #2A7DB3、橙色 #FF8C69、绿色 #6DD5B0、黄色 #FFD166、红色 #FF6B6B。不要使用 emoji、卡通插画、复杂渐变、厚重阴影或照片质感。

资产命名建议：图标可按功能命名，例如 icon_weather、icon_traffic、icon_shop、icon_parking、icon_queue、icon_other、icon_location、icon_time、icon_reward、icon_wallet、icon_report、icon_appeal。背景图可按页面或模块命名，例如 bg_home_header、bg_empty_state、bg_wallet_balance。

交互视觉要求：
所有按钮要有明确主次。底部固定操作栏必须避开手机安全区。列表页面要适合快速扫读。表单页面要分组清楚。界面要像真实 App 截图，而不是线框图或概念海报。
```

## 1. 首页 pages/index/index

```text
基于“问当地”统一 UI 风格，设计微信小程序【首页】。

页面目标：
让用户一打开就明白两件事：可以发布本地实时求助，也可以去任务大厅帮助别人接单赚钱。首页不是营销页，而是实际可操作的产品入口。

页面顶部：
保留微信小程序原生导航栏，标题为“子凡问问-问当地”。页面内容顶部是品牌欢迎区，左侧显示“问当地”品牌标题，右侧可有轻量品牌图标。未登录时显示欢迎插画位、标题“欢迎来到问当地”、副文案“登录后即可发布求助或帮助他人”，并提供主按钮“微信一键登录”。

核心操作区：
设计一个醒目的发布求助入口卡片。卡片标题“发布求助”，副文案“想问什么，当地人帮你解答”。卡片内展示三个短标签：“实时位置”“多种类型”“快速响应”。点击整张卡片进入发布页。卡片整体要像产品核心入口，不要像广告横幅。

积分提示：
如果用户积分不足，发布入口下方可以显示一条轻量提示：“积分不足？去 签到 获取积分”，其中“签到”为可点击文字。提示不要占据太多高度。

附近任务预览：
模块标题为“附近求助”，旁边显示范围标签“5公里内”，右侧有“查看全部”。未登录时显示登录后查看附近任务的空状态，提示“成都地区的实时求助信息”。已登录时展示附近任务卡片，卡片包含类型、问题摘要、金额或积分、地点、距离和紧急/时间信息。示例：类型“店铺营业”，悬赏“¥8”，问题“帮我确认春熙路这家咖啡店现在还营业吗？”，地点“春熙路 · 1.2km”，时间“12 分钟前”。另一个示例：类型“停车空位”，悬赏“¥12”，问题“太古里地下停车场现在还有空位吗？”，地点“太古里 · 3.6km”，时间“刚刚发布”。

底部 Tab：
包含：首页、大厅、消息、我的。当前选中“首页”，选中颜色为 #5DB8E6，其他为 #8FA8B8。图标为统一线性风格。

视觉重点：
第一眼看到“发布求助”，第二眼看到“附近求助”。首页当前以成都起步和 5 公里附近任务为口径，不要设计成城市切换型首页，不要加入快捷类型网格。
```

## 2. 发布求助页 pages/publish/publish

```text
基于“问当地”统一 UI 风格，设计微信小程序【发布求助页】。

页面目标：
帮助用户快速发布一个带悬赏的本地实时求助。页面要让用户觉得填写简单、金额清楚、支付可信。

页面顶部：
保留微信小程序原生导航栏，标题为“发布求助”。页面主体从“求助地点”开始，不需要额外顶部提示条。

表单结构：
采用纵向表单分组。每个分组使用白色卡片，卡片圆角 16rpx，间距清楚。不要把所有字段堆在一个大表单里。

分组 1 求助地点：
标题“求助地点”，副文案“选择你想了解的位置”。显示地点选择卡片，未选中时显示“请选择地点”“在地图上选择你想了解的位置”，右侧为箭头；已选中时显示地点名称和地址，并显示“修改”。下方可以展示“常用地点”快捷 chips，当前口径以成都常用地点为示例。

分组 2 求助类型：
标题“求助类型”，副文案“选择你想了解的信息类型”。使用图标选择项展示“实时天气”“道路拥堵”“店铺营业”“停车空位”“排队情况”“其他”。选中项有对勾和浅蓝底。

分组 3 详细描述：
标题“详细描述”，标注“选填，让帮助者更了解你的需求”。多行输入框，占位示例：“例如：这家店现在还营业吗？门口排队大概多久？”下方显示字数提示“0/500”。

分组 4 添加图片：
标题“添加图片”，标注“选填，最多3张，每张不超过10MB”。展示图片缩略图和添加方块，每张图片有删除按钮。

分组 5 有效时间：
标题“有效时间”，副文案“超时自动取消，金额原路退回”。使用 chips 展示 30分钟、1小时、2小时、4小时、12小时、24小时，可有“推荐”标签。

分组 6 悬赏金额：
标题“悬赏金额”，副文案“设置悬赏，吸引更多帮助者”。展示推荐金额按钮和自定义金额输入。金额明细区显示“悬赏金额”“平台服务费（百分比）”“帮助者实际获得”，帮助者实际获得用绿色或深色突出。

分组 7 支付方式：
标题“支付方式”。两张横向支付方式卡片，顺序为“余额支付，可用 ¥xx”和“微信支付，即时到账”。余额不足时显示橙色提示“可用余额不足，请选择微信支付”。当前默认支付方式根据余额是否足够自动选择。

平台规则入口：
支付方式下方显示轻量入口“平台规则：……”和箭头，点击后可打开“平台规则说明”弹窗。弹窗包含悬赏支付、平台服务费、取消退款、任务完成、提现规则等条目。

底部固定操作栏：
白色背景，顶部浅分割线。左侧显示“需支付 ¥8.00”，金额橙色加粗。右侧主按钮“立即支付并发布”，处理中显示“处理中...”。

视觉重点：
金额和支付部分要清楚稳重。底部按钮固定但不遮挡内容。不要使用夸张插画，不要复杂背景。
```

## 3. 任务大厅页 pages/task-hall/task-hall

```text
基于“问当地”统一 UI 风格，设计微信小程序【任务大厅页】。

页面目标：
帮助者浏览附近可接任务，快速判断哪个任务值得接。页面应具备较高扫读效率，突出地点、距离、悬赏、剩余时间和问题摘要。

页面顶部：
保留微信小程序原生导航栏，标题为“任务大厅”。页面内容顶部直接进入筛选和任务列表，不需要单独的城市头部。

筛选区域：
顶部第一行是任务类型横向标签栏，包含“全部”“实时天气”“道路拥堵”“店铺营业”“停车空位”“排队情况”“其他”。选中项使用天蓝色高亮，类型顺序可根据用户帮助者资料偏好排序。

次级筛选栏：
类型标签下方放两个胶囊筛选入口：“距离最近/金额最高/最新发布”等排序方式，以及“距离范围”。点击后以底部弹窗形式展示选项，弹窗标题分别为“排序方式”“距离范围”，选中项显示对勾。

帮助者资料提示：
如果用户未完善帮助者资料，在列表上方显示浅蓝提示卡：“完善帮助者资料”，说明“设置常去地点和可帮助类型，获取更精准的任务推荐”，右侧按钮“去完善”，可关闭。

任务列表：
筛选区下方显示统计文字，例如“附近共有 18 个求助”。展示 3 张任务卡片，每张卡片结构统一：左侧有类型色条，顶部显示类型图标和类型名称，右侧显示悬赏金额；中部显示问题描述；底部显示地点、距离、时间或剩余时间；右下角主按钮“接单”。卡片 1：类型“店铺营业”，悬赏“¥8”，问题“帮我看一下春熙路这家咖啡店现在还营业吗？”，地点“春熙路 · 1.2km”，时间“12 分钟前”。卡片 2：类型“道路拥堵”，悬赏“¥15”，问题“二环高架往成渝立交方向现在堵不堵？”，地点“二环高架 · 2.8km”。卡片 3：类型“停车空位”，悬赏“¥12”，问题“太古里地下停车场现在还有空位吗？”，地点“太古里 · 3.6km”。

空状态：
没有任务时显示“附近暂无求助任务”，说明“下拉页面刷新，或稍后再来看看”。如果当前不是全部类型，显示按钮“查看全部类型”。

底部 Tab：
包含：首页、大厅、消息、我的。当前选中“大厅”。

视觉重点：
这是任务浏览页，不是地图页。信息密度中等偏高，但要干净易扫读。类型筛选、排序弹窗和距离弹窗都要清楚，不能设计成四项固定筛选栏。
```

## 4. 任务详情页 pages/task-detail/task-detail

```text
基于“问当地”统一 UI 风格，设计微信小程序【任务详情页】。

页面目标：
让用户清楚了解任务内容、地点、悬赏、状态和下一步操作。帮助者可以判断是否接单；求助者可以查看进度；进行中任务可以进入聊天、确认完成或举报。

页面顶部：
保留微信小程序原生导航栏，标题为“任务详情”。

顶部状态卡：
白色卡片，圆角 16rpx。左上角显示状态标签“待匹配”，使用浅蓝或黄色底。右上角显示悬赏金额“¥8”，橙色大号加粗。主标题为问题摘要：“帮我看一下春熙路这家咖啡店现在还营业吗？”下方显示类型标签“店铺营业”和发布时间“12 分钟前发布”。

地点信息卡：
标题“求助位置”。显示定位图标 + 地点名称，例如“春熙路 88 号”，下方弱文字显示地址或距离，右侧用箭头表示可打开地图查看。不要放复杂真实地图。

任务说明卡：
标题“求助内容”。正文展示发布者的详细描述，例如“我准备半小时后过去，想确认现在是否还营业，门口是否需要排队。”下方如果有图片，显示“求助图片”缩略图区域；无图时不强行展示图片区。

时间与规则卡：
任务信息区：
展示任务单号、求助者和帮助者信息。求助者卡显示昵称、头像或“我自己”；帮助者未接单时显示“等待有缘人接单...”，已接单时显示帮助者信息。下方显示发布时间、接单时间、完成时间等时间字段，有值才展示。

安全提示区：
不同状态下显示状态提示：待匹配可展示“等待有缘人接单...”；审核中显示“任务正在审核中，暂不可操作”；已完成显示“任务已完成”；已取消显示“任务已取消”。

底部固定操作栏：
根据角色和任务状态显示按钮。待匹配帮助者视角显示主按钮“立即接单（+¥xx）”；进行中求助者视角显示“进入聊天”“确认完成”；进行中双方都可从聊天或详情进入举报；待匹配发布者可显示“取消任务”。审核中、已完成、已取消状态不显示主操作，只显示状态提示或查看入口。

视觉重点：
任务详情页要比列表页更稳重。当前实现重点是求助内容、图片、位置、任务单号、双方角色和状态操作，不要设计独立流程步骤条。
```

## 5. 聊天页 pages/chat/chat

```text
基于“问当地”统一 UI 风格，设计微信小程序【聊天页】。

页面目标：
这是任务双方围绕某一笔求助进行沟通的页面，不是普通社交聊天。用户进入后必须始终知道：当前聊的是哪一个任务、任务状态是什么、悬赏金额是多少、下一步应该做什么。
```
