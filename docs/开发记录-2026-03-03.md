## 十一、重要Bug修复记录

### 🚨 聊天历史消息查询Bug（数据串聊风险）

**发现时间**: 2026-03-03
**严重程度**: 严重 - 可能导致用户看到其他聊天的消息
**影响范围**: `wdd-chat` 云函数的 `getMessages` 接口

#### 问题描述
加载历史消息时，如果传入 `beforeTime` 参数（下拉刷新加载更多），查询条件会丢失 `need_id` 过滤，导致返回其他任务的消息。

#### 根本原因
云开发数据库SDK中，连续调用 `.where()` 会**覆盖**之前的条件，而不是合并。

```javascript
// ❌ 错误代码（Bug）
let query = db.collection('wdd-messages').where({
  need_id: needId  // 这个条件会被覆盖
})
if (beforeTime) {
  query = query.where({
    create_time: _.lt(new Date(beforeTime))  // 覆盖！need_id丢失
  })
}
```

#### 修复方案
将条件合并到单个 `where` 对象中：

```javascript
// ✅ 正确代码（修复后）
let query
if (beforeTime) {
  query = db.collection('wdd-messages').where({
    need_id: needId,
    create_time: _.lt(new Date(beforeTime))
  })
} else {
  query = db.collection('wdd-messages').where({
    need_id: needId
  })
}
```

#### 前端防护措施
为防止类似问题，在 `chat.js` 中增加多层防护：

1. **页面卸载时清理状态**
```javascript
onUnload() {
  // 停止监听器
  if (this.messageListener) {
    this.messageListener.close()
    this.messageListener = null
  }
  // 清空已处理消息集合
  this.processedMessageIds.clear()
}
```

2. **监听器回调中验证任务ID**
```javascript
// 确保消息属于当前任务
const currentNeedId = this.data.task._id
if (!currentNeedId || currentNeedId !== needId) {
  console.log('页面已切换，忽略旧监听器消息')
  return
}
```

3. **重启监听时重置状态**
```javascript
// 停止旧监听时清空已处理集合
if (this.messageListener) {
  this.messageListener.close()
  this.processedMessageIds.clear()  // 清空避免ID冲突
}
```

6. **数据库查询**: 云开发SDK中连续 `.where()` 会覆盖而非合并条件，需使用单个对象传入所有条件
