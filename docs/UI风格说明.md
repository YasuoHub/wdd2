# 问当地 UI 风格说明

**更新日期**：2026-06-16
**适用范围**：微信小程序前端页面与公共组件
**当前风格关键词**：微信城市服务、真实小程序、清爽可信、地点感强、交易清楚、信息高效、白底浅灰、克制蓝绿、低装饰、专业但亲和。

## 1. 设计目标

“问当地”涉及悬赏、接单、聊天确认、结算提现、举报申诉等真实交易场景。界面首先要让用户相信：信息清楚、规则明确、资金流向可靠、每一步都知道自己在做什么。

核心目标：

1. 信息优先：地点、状态、金额、剩余时间要容易扫读。
2. 操作明确：主按钮突出，次要按钮克制，危险操作必须可识别。
3. 可信克制：避免营销页、抢单平台、金融理财产品的强刺激感。
4. 地点感强：保留地图、距离、城市服务线索，但作为辅助信息，不做大面积装饰。
5. 微信原生感：像成熟小程序，而不是临时 demo。

## 2. 全局视觉方向

全局设计令牌定义在 `miniprogram/app.wxss`。当前视觉基准为：

| 维度 | 要求 |
| --- | --- |
| 背景 | 页面使用浅灰白背景，避免大面积渐变 |
| 卡片 | 白底、细边框、轻阴影、克制圆角 |
| 主色 | 克制蓝，用于主操作、选中态、链接 |
| 辅助色 | 绿色表示成功，橙色表示交易/悬赏/提醒，红色表示风险 |
| 图标 | 使用线性图标或现有 icon 组件，不使用表情作为结构图标 |
| 动效 | 只保留必要反馈，避免漂浮、发光、强弹跳 |

| 类型 | 变量 | 色值 | 用途 |

| 深主色 | `--sky-deep` | `#135FA7` | 强调文字、重要入口 |
| 浅主色 | `--sky-pale` | `#EEF7FD` | 弱背景、位置提示 |
| 交易色 | `--vitality-orange` | `#D96A22` | 悬赏金额、支付、提醒 |

## 4. 组件规范

### 按钮

主按钮使用纯主色背景和白字；次要按钮使用白底、细边框和主色文字；交易相关按钮使用交易橙；危险操作使用红色。按钮文案要短，例如“立即发布”“确认完成”“提交申诉”。

### 卡片

任务、钱包、工单、举报、申诉卡片统一使用白底、细边框、轻阴影。任务卡片必须优先呈现：类型、地点/距离、问题摘要、悬赏金额、状态或剩余时间。

### 标签

标签只表达类型或状态，不做装饰。颜色含义保持稳定：

| 含义 | 推荐颜色 |
| --- | --- |
| 待匹配/待处理 | 浅蓝或浅黄 |
| 进行中 | 主蓝 |
| 已完成 | 绿色 |
| 审核中/提醒 | 橙色 |
| 争议/举报/错误 | 红色 |
| 已取消/已关闭 | 灰色 |

### 弹窗和底部面板

弹窗使用白底、清楚标题、明确取消/确认动作。遮罩不宜过浅，保证前景可读。涉及支付、提现、取消、举报、申诉、裁定的动作必须保留二次确认。

## 5. 页面重点

首页突出“发布求助”和“去大厅接单”两个入口，保留附近实时信息感，但不过度装饰。

发布页是转化核心，表单分组必须清楚：问什么、问哪里、给多少悬赏、多久有效、是否补充图片。支付前要清楚展示金额和平台规则。

任务大厅强调快速扫读。筛选栏不要过重，任务卡片优先展示地点、距离、悬赏、摘要和状态。

聊天页要像围绕任务的沟通工具，顶部保留任务摘要和状态，输入区保持微信式克制。

我的、钱包、提现页面要更稳重，金额、流水、提现状态必须清楚，避免强刺激色彩造成误解。

举报、申诉、工单页面强调证据、事实和处理进度，减少情绪化表达。

## 6. 交付检查

1. 不改变页面字段、业务状态、弹窗、角色视角和跳转关系。
2. 所有主要按钮文字对比充足，触控区域不小于 44pt。
3. 页面不出现横向滚动，不让底部固定栏遮挡内容。
4. 空状态、加载状态、错误状态与主页面风格一致。
5. 在微信开发者工具中检查首页、发布、大厅、聊天、我的、钱包、举报/申诉流程。
