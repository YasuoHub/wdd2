# 问当地 UI 风格说明

**更新日期**：2026-05-14  
**适用范围**：微信小程序前端页面与组件  
**当前风格关键词**：青春活力、天蓝主调、亲和、轻量、可信

## 1. 设计目标

“问当地”不是严肃办公工具，也不是纯社交娱乐产品。它需要让用户感觉：

1. 轻松：发布一个问题不应有压力。
2. 可信：涉及悬赏、接单和资金，界面必须清楚稳重。
3. 快速：用户是为了即时信息而来，操作路径要短。
4. 友好：求助和帮助都带有人情味，不要像冷冰冰的订单系统。

## 2. 整体视觉方向

当前全局样式定义在 `miniprogram/app.wxss`，采用天蓝色作为主色，并配合橙色、薄荷绿、黄色等少量活力色。

整体观感应保持：

| 维度 | 要求 |
| --- | --- |
| 气质 | 清爽、阳光、有帮助感 |
| 信息密度 | 中等，任务卡片要易扫读 |
| 按钮 | 圆润、明确、主次分明 |
| 卡片 | 白色或半透明白色，轻阴影 |
| 反馈 | 操作成功、加载、空状态、错误状态都要有清楚提示 |

## 3. 颜色规范

### 3.1 主色

| 名称 | 变量 | 色值 | 用途 |
| --- | --- | --- | --- |
| 天蓝主色 | `--sky-primary` | `#5DB8E6` | 主按钮、选中态、重点图标 |
| 天蓝浅色 | `--sky-light` | `#7EC8E8` | 渐变、辅助高亮 |
| 天蓝柔色 | `--sky-soft` | `#A8D8F0` | 背景、弱强调 |
| 天蓝深色 | `--sky-deep` | `#2A7DB3` | 链接、标题强调 |

### 3.2 辅助色

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 活力橙 | `#FF8C69` | 支付、悬赏、重要提醒 |
| 珊瑚色 | `#FF9A8B` | 橙色渐变 |
| 薄荷绿 | `#6DD5B0` | 完成、成功、收益 |
| 柔黄色 | `#FFD166` | 积分、提示、等待 |
| 危险红 | `#FF6B6B` | 删除、举报、错误 |

### 3.3 文字色

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 主文字 | `#2C3E50` | 标题和主要内容 |
| 次文字 | `#4A6572` | 普通说明 |
| 弱文字 | `#6B8BA4` | 时间、地点、辅助信息 |
| 浅文字 | `#8FA8B8` | 占位、未选中 Tab |

## 4. 字体与排版

默认字体使用系统字体：

```css
-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC",
"Hiragino Sans GB", "Microsoft YaHei", sans-serif
```

排版建议：

| 类型 | 建议 |
| --- | --- |
| 页面标题 | 36-44rpx，粗体 |
| 模块标题 | 30-34rpx，半粗 |
| 正文 | 28rpx |
| 辅助文字 | 22-26rpx |
| 金额 | 可适当加粗，颜色用橙色或深色 |
| 标签文字 | 22rpx 左右 |

页面内文字不要过度使用大字号。任务列表、钱包、工单等操作型页面应优先保证可读性和扫描效率。

## 5. 圆角、阴影与间距

当前全局变量：

| 类型 | 变量 | 用途 |
| --- | --- | --- |
| 小圆角 | `--radius-xs`、`--radius-sm` | 标签、输入框 |
| 中圆角 | `--radius-md`、`--radius-lg` | 卡片、弹层 |
| 大圆角 | `--radius-xl`、`--radius-full` | 主按钮、胶囊标签 |
| 轻阴影 | `--shadow-sm` | 普通卡片 |
| 中阴影 | `--shadow-md` | 强调模块 |
| 彩色阴影 | `--shadow-sky`、`--shadow-orange` | 主按钮、悬赏按钮 |

间距建议：

1. 页面左右边距保持 24-32rpx。
2. 卡片内边距保持 28-32rpx。
3. 列表项之间保持 20-28rpx。
4. 主操作按钮与内容之间保持足够呼吸感，避免误触。

## 6. 组件风格

### 6.1 按钮

| 按钮 | 样式 | 使用场景 |
| --- | --- | --- |
| 主按钮 | 天蓝渐变、白字、圆角 | 发布、接单、确认、提交 |
| 次按钮 | 白色/玻璃感、蓝色文字 | 返回、筛选、查看 |
| 警示按钮 | 红色或红色描边 | 举报、撤销、删除 |
| 收益按钮 | 薄荷绿 | 提现、收益相关确认 |
| 支付按钮 | 橙色 | 支付、悬赏确认 |

按钮文案要短，例如“立即发布”“确认完成”“提交申诉”。不要在按钮里写完整说明句。

### 6.2 卡片

任务卡片应包含：

1. 类型标签。
2. 地点和距离。
3. 问题摘要。
4. 悬赏金额。
5. 状态或剩余时间。

钱包、评价、工单卡片应突出当前状态，避免用户不知道下一步能做什么。

### 6.3 标签

标签用于表达状态和类型。建议固定颜色含义：

| 含义 | 推荐颜色 |
| --- | --- |
| 待匹配 | 黄色或天蓝浅色 |
| 进行中 | 天蓝色 |
| 已完成 | 薄荷绿 |
| 已取消 | 灰色 |
| 审核中 | 橙色 |
| 举报/风险 | 红色 |

## 7. 关键页面风格

### 7.1 首页

首页要让新用户马上知道可以做两件事：发布求助、去大厅接单。首屏应突出发布入口和附近实时信息感。

### 7.2 发布页

发布页是转化核心。表单应分组清晰：

1. 问什么。
2. 问哪里。
3. 给多少悬赏。
4. 多久有效。
5. 是否补充图片。

支付前必须清楚展示金额和规则。

### 7.3 任务大厅

任务大厅要适合快速扫读。任务卡片优先展示地点、距离、悬赏、问题摘要和状态。筛选项不宜占据过多屏幕高度。

### 7.4 聊天页

聊天页要像一个围绕任务的沟通工具，而不是普通聊天软件。顶部应保留任务摘要和任务状态，让双方知道当前聊的是哪一单。

### 7.5 我的与钱包

“我的”页面承担用户资产和信任入口。钱包页必须比普通页面更稳重，金额、提现状态、流水要清晰，避免误解。

### 7.6 举报、申诉、工单

争议页面要减少情绪化表达，强调证据、事实和处理进度。颜色可使用橙色和红色提示风险，但不能让页面显得失控。

## 8. 空状态、错误状态与加载状态

项目已有 `loading`、`empty`、`error` 组件。使用建议：

| 状态 | 表达 |
| --- | --- |
| 加载中 | 告诉用户正在获取，不要让页面空白 |
| 无数据 | 告诉用户没有数据，并给一个下一步入口 |
| 错误 | 说明失败原因，提供重试 |
| 无权限 | 温和说明需要登录、授权位置或不是任务相关人 |

## 9. 交互原则

1. 涉及支付、取消、提现、举报、申诉、裁定，必须二次确认。
2. 涉及位置授权，先说明用途，再请求授权。
3. 表单提交失败时，指出具体字段，不要只说“提交失败”。
4. 操作成功后给明确反馈，并跳转到用户下一步最可能要去的页面。
5. 长列表保留刷新和加载更多反馈。

## 10. 后续视觉优化建议

1. 建立统一状态标签组件，避免每个页面重复写状态颜色。
2. 统一任务卡片结构，让首页、大厅、我的求助、我的任务体验一致。
3. 钱包和工单页面应降低装饰感，提高可信度。
4. 增加一套正式图标资源，替换临时符号和不统一图标。
5. 做一次真机截图检查，覆盖 iPhone 小屏、常规安卓和大屏设备。
