# 微信支付配置指南

> 本文档用于指导将"问当地"小程序从模拟支付切换到真实微信支付。

---

## 一、前置条件确认

在开始前，请确认以下事项已完成：

| 项目 | 状态 | 说明 |
|------|------|------|
| 微信商户号已开通 | 必须 | 已在 pay.weixin.qq.com 注册并完成企业认证 |
| 小程序已关联商户号 | 必须 | 在小程序后台「微信支付」中绑定商户号 |
| 云开发环境已开通 | 必须 | 本项目使用云开发环境 `wdd-2grpiy1r6f9f4cf2` |

## 二、需要准备的材料

### 2.1 商户平台获取（关键）

登录 [微信支付商户平台](https://pay.weixin.qq.com/)，依次完成以下操作：

#### ① 设置 APIv3 密钥

路径：`账户中心 → API安全 → APIv3密钥 → 设置`

- 点击"设置"，输入 32 位随机字符串（可在线生成）
- **务必保存好这个密钥，只显示一次，后续无法查看**
- 建议将密钥保存在安全的地方（如密码管理器）

#### ② 申请 API 证书

路径：`账户中心 → API安全 → API证书 → 申请证书`

- 按指引完成证书申请流程
- 下载后得到三个文件：
  - `apiclient_cert.pem`（商户证书）
  - `apiclient_key.pem`（商户证书私钥）
  - `apiclient_cert.p12`（可选，备用格式）
- **私钥文件（apiclient_key.pem）极其重要，切勿泄露**

#### ③ 记录证书序列号

路径：`账户中心 → API安全 → API证书 → 查看证书`

- 找到你刚申请的证书，记录其**证书序列号**
- 格式类似：`4433435CXXXXXXXXXXXXXXXXXXXXX`

#### ④ 记录商户号

路径：`账户中心 → 商户信息 → 商户号`

- 记录你的 10 位数字商户号，如 `1234567890`

### 2.2 材料清单汇总

准备完成后，你应该有以下信息：

```
商户号（mchid）: 1234567890
APIv3 密钥    : xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx（32位）
证书序列号    : 4433435CXXXXXXXXXXXXXXXXXXXXX
商户证书文件  : apiclient_cert.pem
商户私钥文件  : apiclient_key.pem
```

## 三、云开发控制台配置

### 3.1 绑定商户号

1. 打开**微信开发者工具**
2. 点击左上角的「云开发」按钮
3. 进入「设置 → 全局设置 → 微信支付」
4. 点击「添加商户号」
5. 输入商户号，按提示完成绑定

### 3.2 配置 APIv3 密钥和证书

在云开发控制台：

1. 进入「设置 → 全局设置 → 微信支付」
2. 找到你绑定的商户号
3. 点击「配置」
4. 填入以下内容：
   - **APIv3 密钥**：粘贴 32 位密钥
   - **商户证书**：上传 `apiclient_cert.pem` 文件内容
   - **商户私钥**：上传 `apiclient_key.pem` 文件内容
5. 保存配置

> 注意：云开发控制台可能需要你粘贴证书文件的**文本内容**，而非上传文件。请用文本编辑器打开 `.pem` 文件，复制全部内容粘贴。

## 四、代码启用真实支付

配置完成后，修改代码开启真实支付：

### 4.1 修改支付云函数

文件：`cloudfunctions/wdd-payment/index.js`

找到第 14 行：

```javascript
const MOCK_PAYMENT = true
```

改为：

```javascript
const MOCK_PAYMENT = false
```

### 4.2 部署云函数

在微信开发者工具中：

1. 右键 `cloudfunctions/wdd-payment` → 「创建并部署：云端安装依赖」
2. 等待部署完成

## 五、配置验证（测试支付）

### 5.1 发起一笔测试支付

1. 在小程序中发布一个求助任务
2. 填写悬赏金额（建议 0.01 元用于测试）
3. 点击支付
4. 观察是否调起微信支付弹窗

### 5.2 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 支付弹窗未出现 | 商户号未绑定 | 检查云开发控制台绑定状态 |
| 提示"签名错误" | APIv3 密钥不正确 | 重新核对密钥，注意大小写 |
| 提示"证书错误" | 证书文件内容不对 | 重新复制粘贴证书完整内容 |
| 支付成功但任务未创建 | 支付回调未正确处理 | 检查云函数日志 |

## 六、提现功能配置（可选，可后续开通）

真实提现需要开通**商家转账到零钱**功能。

### 6.1 开通条件

微信支付的要求（2024年新规）：

- 商户入驻满 **90 天**
- 连续交易满 **30 天**
- 完成风险合规审核

### 6.2 开通路径

1. 登录 [微信支付商户平台](https://pay.weixin.qq.com/)
2. `产品中心 → 商家转账到零钱 → 申请开通`
3. 按指引提交材料，等待审核（通常 1-3 个工作日）

### 6.3 未开通前的过渡方案

如果暂时不满足开通条件，可以采用**人工审核提现**：

1. 用户在小程序提交提现申请
2. 管理员在后台（或数据库）查看申请记录
3. 人工通过微信/支付宝/银行卡转账给用户
4. 在数据库中标记该提现已完成

> 代码中已预留 `adminApprove` 接口，支持管理员手动审核通过。

## 七、安全提醒

1. **私钥文件（apiclient_key.pem）切勿上传到代码仓库**
2. **APIv3 密钥不要硬编码在代码中**（云开发已封装，无需在代码中写密钥）
3. **支付相关云函数建议开启调用统计**，及时发现异常
4. **定期在商户平台查看交易流水**，核对账目
