# 支付字段说明

发布任务的支付订单 `wdd-payment-orders` 与任务 `wdd-needs` 使用相同的金额结构：

| 字段 | 含义 |
| --- | --- |
| `total_amount` | 本次悬赏支付总额 |
| `deduction_amount` | 平台抵扣金支付金额 |
| `balance_amount` | 用户余额支付金额 |
| `wechat_amount` | 微信支付金额 |
| `payment_methods` | 实际参与支付的方式，如 `['deduction', 'balance', 'wechat']` |

必须满足：

```text
total_amount = deduction_amount + balance_amount + wechat_amount
```

## 分摊顺序

用户可独立选择平台抵扣金、余额和微信支付。系统只使用用户选中的方式，并按以下顺序分摊：

1. 平台抵扣金
2. 余额
3. 微信支付剩余金额

后端会根据数据库中的实时可用资金重新计算明细，不信任前端提交的金额。

## 资金处理

- 发布时：平台抵扣金和余额先冻结，微信部分即时支付。
- 正常完成：扣除已冻结的平台抵扣金和余额，微信部分不重复处理。
- 全额取消：平台抵扣金和余额解冻，微信部分原路退款。
- 部分完成：按原支付构成比例计算退款明细；未退款部分作为实际消费，退款部分按原渠道退回。

退款订单使用以下字段记录明细：

| 字段 | 含义 |
| --- | --- |
| `total_refund_amount` | 退款总额 |
| `deduction_refund_amount` | 平台抵扣金退款金额 |
| `balance_refund_amount` | 余额退款金额 |
| `wechat_refund_amount` | 微信原路退款金额 |
