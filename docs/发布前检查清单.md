# 发布前检查清单

**更新日期**：2026-06-02  
**适用范围**：公开发布前的产品、测试、运营、开发联合验收。

## 1. 当前发布边界

发布前版本只验证核心交易闭环：

1. 求助者发布真实悬赏任务。
2. 帮助者在任务大厅主动浏览并接单。
3. 双方通过任务聊天完成信息确认。
4. 求助者确认完成，系统完成结算。
5. 出现争议时，通过举报、申诉和客服工单裁定。

当前不验证、不承诺：

1. 自动派单、人工派单、顺序轮询派发。
2. 消息置顶、消息删除、会话归档。
3. 积分支付、积分提现、积分抵现金。
4. 积分权益兑换。

## 2. 真实数据兜底清理

发布前必须确认：接口失败、无数据、网络异常时，用户只能看到错误状态、空状态或重试入口，不能看到模拟业务数据。

| 检查位置 | 当前风险 | 发布前处理标准 |
| --- | --- | --- |
| `miniprogram/pages/my-needs/my-needs.js` | 加载失败时可能展示模拟求助列表 | 移除模拟列表兜底，改为错误组件或空状态 |
| `miniprogram/pages/my-tasks/my-tasks.js` | 加载失败时可能展示模拟接单列表 | 移除模拟列表兜底，改为错误组件或空状态 |
| `miniprogram/pages/rating/rating.js` | 任务信息加载失败时可能展示测试用户和测试任务 | 阻断评价提交，并提示重新进入任务 |
| `miniprogram/pages/my/my.js` | 任务数量加载失败时归零 | 可接受，但要避免让用户误以为真实统计被清空 |

验收口径：搜索 `模拟数据`、`测试用户`、`mockData`，确认不会进入真实用户路径。

## 3. 主流程测试

| 链路 | 验收点 |
| --- | --- |
| 新用户登录 | 能创建用户，积分文案不暗示可提现或抵现金 |
| 位置授权 | 同意、拒绝、重新授权都有清晰反馈 |
| 发布任务 | 使用真实地点、真实金额、真实支付方式创建任务 |
| 大厅接单 | 帮助者主动进入大厅接单，不等待系统派单 |
| 并发接单 | 同一个任务只能被一个帮助者接走 |
| 聊天 | 文字、图片、未读提醒、内容安全审核正常 |
| 确认完成 | 帮助者收入入账，求助者支出记录正确 |
| 待匹配取消 | 任务取消后金额按支付方式退回或解冻 |
| 进行中超时 | 按当前规则自动完成、提醒或进入异常处理 |
| 评价 | 只能评价真实任务，不允许测试任务兜底 |

## 4. 资金测试

| 链路 | 验收点 |
| --- | --- |
| 微信支付 | 支付成功后任务进入待匹配 |
| 余额支付 | 发布时冻结，完成时扣款，取消时解冻 |
| 退款 | 微信支付和余额支付都能正确处理取消/裁定退款 |
| 提现 | 小额即时提现、大额审批提现按配置执行 |
| 提现失败 | 用户能看到可理解的状态，后台有审计记录 |
| 收支明细 | 每一笔收入、支出、冻结、解冻都有对应记录 |

## 5. 争议测试

| 链路 | 验收点 |
| --- | --- |
| 举报 | 任务进入 `breaking`，聊天输入被锁定 |
| 申诉 | 能提交原因和证据，双方状态一致 |
| 补充材料 | 截止时间和补充状态可追踪 |
| 客服裁定取消 | 任务取消，资金退回求助者 |
| 客服裁定完成 | 任务完成，资金结算给帮助者 |
| 客服裁定部分完成 | 按比例分配，记录可追溯 |
