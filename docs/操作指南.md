# 问当地小程序操作指南

**更新日期**：2026-05-14  
**适用对象**：普通用户、帮助者、测试人员、运营人员  
**说明**：本文档描述当前产品的主要操作路径，具体页面名称以小程序当前版本为准。

## 1. 首次使用

1. 打开“问当地”小程序。
2. 按提示完成微信授权登录。
3. 如需查看附近任务或发布定位相关求助，允许小程序获取位置信息。
4. 进入首页后，可选择发布求助或进入任务大厅。

注意：如果用户拒绝位置授权，仍可浏览部分页面，但附近任务、距离排序、地图选点等能力会受影响。

## 2. 发布一个求助

1. 从首页或相关入口进入“发布求助”。
2. 选择求助类型，例如实时天气、道路拥堵、店铺营业、停车空位、排队情况。
3. 选择或填写求助地点。
4. 输入问题描述，尽量写清楚“想知道什么”和“希望对方怎么回答”。
5. 设置悬赏金额。
6. 选择有效期。
7. 按需上传图片。
8. 点击发布并完成微信支付。
9. 支付成功后，任务进入待匹配状态。

建议问题示例：

```text
我 30 分钟后到这个商场，想知道地下停车场现在还有没有空位，入口排队大概多久？
```

## 3. 查看自己的求助

1. 进入“我的”。
2. 点击“我的求助”。
3. 查看不同状态的任务。
4. 点击任务进入详情，可查看接单情况、聊天、完成、取消、评价、举报或申诉入口。

## 4. 帮助别人并接单

1. 进入底部导航“大厅”。
2. 允许位置授权，以便查看附近任务和距离。
3. 按类型、距离或排序筛选任务。
4. 点击感兴趣的任务查看详情。
5. 确认自己能提供有效信息后点击接单。
6. 接单成功后进入聊天，与求助者沟通。
7. 提供信息后等待对方确认完成。

接单前请确认：

1. 自己确实了解该地点或可以快速核实。
2. 能在任务有效时间内回复。
3. 不诱导对方私下交易。
4. 不提供虚假信息。

## 5. 聊天沟通

1. 从任务详情或消息中心进入聊天。
2. 查看顶部任务信息，确认当前沟通的是哪一个任务。
3. 发送文字或图片说明现场情况。
4. 对方的新消息会在“消息”Tab 显示角标。
5. 任务完成后可进入评价。

建议帮助者回答方式：

```text
我现在在店门口，店还开着，门口写的是 22:00 打烊。现在店里大约有 6 桌客人，不用排队。
```

## 6. 完成任务

1. 求助者确认帮助者已经提供有效信息。
2. 在任务详情中点击确认完成。
3. 系统完成结算。
4. 帮助者收入进入钱包余额。
5. 双方可进行评价。

如果双方对结果有争议，不建议直接线下协商，应使用举报或申诉入口保留处理记录。

## 7. 取消任务

待匹配状态下，求助者可取消任务。取消后任务进入已取消状态，已支付金额按退款流程处理。

已接单的任务通常不应由用户随意取消。如果出现争议，应走举报、申诉或客服工单流程。

## 8. 评价

1. 任务完成后进入评价页。
2. 选择星级。
3. 填写评价内容。
4. 提交后评价会进入对方信誉记录。

评价应基于本次任务事实，不发布辱骂、隐私或无关内容。

## 9. 钱包与提现

### 9.1 查看钱包

1. 进入“我的”。
2. 点击“钱包”。
3. 查看可用余额、冻结金额、累计收入、累计提现。
4. 查看余额流水。

### 9.2 发起提现

1. 进入“钱包”。
2. 点击提现。
3. 输入提现金额。
4. 确认后提交。
5. 等待系统处理微信商家转账。

提现可能受最低提现金额、账户状态、微信支付配置和风控限制影响。

## 10. 积分与签到

1. 新用户首次登录会获得积分。
2. 用户可通过签到获得积分。
3. 邀请新用户可能获得邀请积分。
4. 积分用于平台权益体系，不可提现、不可转让，不等同现金。

## 11. 举报

当遇到以下情况，可发起举报：

1. 对方诱导线下私下交易。
2. 对方辱骂、骚扰、人身攻击。
3. 对方虚假承诺、恶意骗单。
4. 对方故意拖延、失联、不配合。
5. 对方提供虚假实时信息。
6. 帮助者定位明显不符。

举报步骤：

1. 从任务详情进入举报页。
2. 选择举报类型。
3. 填写详细原因。
4. 上传证据图片。
5. 提交举报。
6. 任务进入审核中，等待客服处理。

## 12. 申诉

当对任务结果、举报结果、金额结算或对方行为有异议，可发起申诉。

申诉步骤：

1. 从任务详情或申诉入口进入申诉页。
2. 选择申诉类型。
3. 填写申诉理由。
4. 上传证据图片。
5. 提交申诉。
6. 任务进入审核中，等待客服处理。

## 13. 补充材料

如果客服或系统要求补充材料：

1. 进入举报详情或申诉详情。
2. 查看补充材料截止时间。
3. 上传新的图片或填写补充说明。
4. 提交后等待客服继续处理。

超过补充材料期限后，系统可能按已有材料继续裁定。

## 14. 客服/管理员处理工单

客服入口只对配置在 `wdd-config/platform.customer_service_openids` 白名单中的微信 OpenID 展示。未在白名单中的普通用户不会看到客服工单入口，也不能调用客服裁定接口。

客服处理路径：

1. 进入工单列表。
2. 筛选待处理工单。
3. 查看任务信息、举报/申诉内容、双方资料和补充材料。
4. 根据证据选择裁定结果。
5. 填写客服备注。
6. 提交裁定。

裁定结果：

| 裁定 | 适用场景 | 结果 |
| --- | --- | --- |
| 取消任务 | 帮助无效、违规严重、无法继续履约 | 任务取消，金额退回求助者 |
| 完成任务 | 帮助者已有效完成 | 任务完成，金额发给帮助者 |
| 部分完成 | 双方都有一定责任或帮助部分有效 | 按比例分配金额 |

客服处理原则：

1. 优先看证据，不只看单方描述。
2. 金额处理要与裁定理由一致。
3. 严重违规用户可配合封禁或扣信用分。
4. 每次裁定必须留下备注。

## 15. 测试建议

上线前至少测试以下链路：

1. 新用户登录。
2. 位置授权同意和拒绝。
3. 发布任务并支付。
4. 任务大厅筛选和接单。
5. 双方聊天和未读角标。
6. 确认完成和余额入账。
7. 提交评价。
8. 待匹配任务取消和退款。
9. 提现申请和状态查询。
10. 举报、撤销举报、补充材料。
11. 申诉、撤销申诉、补充材料。
12. 客服工单裁定取消、完成、部分完成。
13. 任务超时自动处理。

## 16. 常见问题

### 16.1 为什么看不到附近任务？

可能原因：

1. 未授权位置信息。
2. 附近暂无待匹配任务。
3. 筛选条件过窄。
4. 网络或云函数请求失败。

### 16.2 支付成功但任务没出现怎么办？

先查看“我的求助”和支付订单状态。如果仍异常，需要通过后台查询 `wdd-payment-orders` 和 `wdd-needs`，确认支付确认流程是否完成。

### 16.3 帮助者完成了，但求助者不确认怎么办？

帮助者可保留聊天记录和图片证据，通过申诉或举报入口提交，等待客服处理。

### 16.4 提现失败怎么办？

查看提现记录状态。常见原因包括微信支付商家转账配置不完整、用户账户限制、余额不足、转账接口返回失败等。
